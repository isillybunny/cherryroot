Bloco 1 · Fundamentos

Os Fundamentos

Antes da sintaxe, as ideias. O que é código? O que é um bug? Entender isso primeiro torna tudo mais fácil depois.
ProgramaçãoCódigoBugIDETerminalSintaxe
10 termos · F·01 → F·10começa aqui
ProgramaçãoF·01
"Programar é dizer ao computador o que fazer, passo a passo."
Escrever instruções precisas em uma linguagem que a máquina entende. Todo app, site e jogo nasce de código.
instruçõeslógicaalgoritmo
# Seu primeiro programa
print("Olá, mundo!")
Código é comunicação — com a máquina e com o você do futuro.
CódigoF·02
"Código é o texto de um programa — escrito em uma linguagem de regras estritas."
Arquivos com extensões como .py, .js, .java guardam código-fonte. Ferramentas como compiladores e interpretadores convertem isso em instruções da máquina.
source.py.jsarquivo
name = "Cherry"
print(f"Olá, {name}!")
Código é texto. Só isso. Arquivos de texto seguindo regras.
Compilador vs InterpretadorF·03
"Duas formas de transformar código em algo que o computador executa."
Compilador traduz tudo antes de rodar (C, Java). Interpretador lê e executa linha por linha (Python, JS).
compileinterpretbytecode
# Compilado
javac Main.java && java Main
# Interpretado
python main.py
Compilado roda rápido; interpretado é flexível. Cada um tem seu lugar.
BugF·04
"Bug é quando o código não faz o que você queria."
Erro de lógica, sintaxe ou comportamento inesperado. O nome veio de uma mariposa real presa num computador em 1947.
errofalhaglitch
# Bug clássico: off-by-one
for i in range(10):
    print(i)  # 0..9, não 1..10
Todo programador tem bugs. Os bons aprendem a achá-los rápido.
DebugF·05
"Debug é o processo de achar e corrigir bugs."
Ler logs, usar prints, breakpoints, testar hipóteses. A maior parte do trabalho é entender o problema, não escrever a correção.
debuggerbreakpointlog
def soma(a, b):
    print(f"a={a}, b={b}")  # debug
    return a + b
Não chute. Confirme. Cada print é uma pergunta com resposta.
IDEF·06
"IDE é um ambiente integrado pra escrever, rodar e debugar código."
Junta editor, terminal, debugger, controle de versão e plugins. VS Code, IntelliJ, PyCharm são os populares.
VS CodeIntelliJplugin
# Abrindo o projeto na IDE
code .          # VS Code
idea .          # IntelliJ
A melhor IDE é a que você domina. Aprenda os atalhos.
TerminalF·07
"Terminal é a interface de texto pra controlar o computador digitando comandos."
Você digita, o shell interpreta, o OS executa. Toda ferramenta séria de dev passa por aqui em algum momento — incluindo o linter.
shellbashCLIprompt
ls -la          # lista arquivos
cd projetos/    # entra na pasta
git status      # mostra mudanças
Terminal não é hostil — é direto. Vale a pena dominar.
SintaxeF·08
"Sintaxe é a gramática estrita de uma linguagem de programação."
Pontos, vírgulas, indentação, parênteses. Uma vírgula faltando e o programa não roda. O parser é inflexível.
parsergramáticaindent
# Python exige indentação
if x > 0:
    print("positivo")  # 4 espaços
Computador não perdoa sintaxe. Vírgula errada = programa quebrado.
RuntimeF·09
"Runtime é o momento em que o código está rodando de verdade."
Diferente da compilação, runtime é quando variáveis ganham valor, funções são chamadas e certos erros só aparecem.
execuçãoJVMV8
x = input("idade: ")     # runtime: lê
print(int(x) + 10)       # runtime: calcula
Erros de runtime só aparecem quando o código roda. Teste sempre.
ParadigmaF·10
"Paradigma é um estilo de programar — uma forma de organizar a lógica."
Imperativo (passos), declarativo (o quê), orientado a objetos, funcional. Linguagens modernas misturam paradigmas.
OOPfuncionalimperativo
# Imperativo
total = 0
for x in [1,2,3]: total += x
# Funcional
total = sum([1,2,3])
Paradigma molda como você pensa. Conhecer vários expande seu repertório.

Bloco 2 · Tipos & Variáveis

Containers de Dados

Toda informação que seu programa manipula vive em uma variável de algum tipo. Strings, números, listas, mapas — saber escolher é metade da batalha.
VariávelStringIntFloatBoolListDict
12 termos · T·01 → T·12guarda valores
VariávelT·01
"Variável é um nome que guarda um valor pra você usar depois."
Você declara, atribui um valor, e usa pelo nome. O valor pode mudar ao longo do programa.
name=storage
age = 25
name = "Cherry"
print(name, age)
O nome importa. x não diz nada; user_age diz tudo.
Tipo de DadoT·02
"Tipo é a categoria do valor — número, texto, booleano, lista..."
O tipo decide o que você pode fazer com o valor. Somar strings concatena; somar números faz aritmética.
intstrbooltype()
type(42)         # int
type("oi")       # str
type([1,2])      # list
Misturar tipos sem cuidado é fonte clássica de bugs. "1" + 1 quebra.
StringT·03
"String é texto — uma sequência de caracteres entre aspas."
Pode usar aspas simples, duplas ou triplas. Strings têm muitos métodos: upper, lower, split, replace, format.
"texto"f-string.split()
msg = "Olá, Cherry"
print(msg.upper())
print(len(msg))   # 11
Strings são imutáveis. Modificar cria uma nova string, não muda a antiga.
InteiroT·04
"Número inteiro — sem decimais, positivo, negativo ou zero."
Usado pra contar, indexar, IDs. Em Python o int não tem limite; em Java tem (32 bits). Veja também Float.
int42-7
qty = 100
total = qty * 3
print(total)   # 300
Divisão de inteiros em Java trunca: 7/2 = 3, não 3.5.
FloatT·05
"Float é número com casas decimais — usado pra medidas, preços, percentuais."
Internamente é binário, então 0.1 + 0.2 ≠ 0.3 exatamente. Cuidado em comparações financeiras.
float3.14decimal
pi = 3.14159
preco = 19.90
print(0.1 + 0.2)  # 0.30000000000000004
Pra dinheiro, prefira Decimal, nunca float. Precisão importa.
BooleanoT·06
"Booleano só tem dois valores: True e False."
Base de toda decisão no código. Toda condição (if, while) avalia uma expressão pra booleano.
TrueFalseand/or/not
logged_in = True
if logged_in:
    print("Bem-vindo")
Tudo no fim vira True ou False. O computador só sabe decidir assim.
Null / NoneT·07
"Representação da ausência de valor — não é zero, não é vazio, é nada."
Python usa None, JS usa null (e undefined), Java usa null. Esquecer disso causa NullPointerException.
NonenullNPE
user = None
if user is None:
    print("sem usuário")
Sempre cheque None antes de usar. NullPointerException é o bug mais comum do mundo.
Lista / ArrayT·08
"Coleção ordenada de valores — acessada por índice começando em 0."
Em Python chama list e é dinâmica. Em Java, array tem tamanho fixo; ArrayList é dinâmica.
[]indexappend
nums = [1, 2, 3, 4]
nums.append(5)
print(nums[0])    # 1
Índice começa em 0. O último item é lista[len-1] ou lista[-1].
Dict / Object / MapT·09
"Coleção de pares chave-valor — busca rápida pela chave."
Em Python, dict. Em JS, object ou Map. Em Java, HashMap. Ideal pra modelar entidades. Internamente é uma hash table.
{}key:valueHashMap
user = {"nome": "Cherry", "idade": 25}
print(user["nome"])
Lista é por ordem; dict é por nome. Use dict quando a chave faz sentido.
ConstanteT·10
"Constante é uma variável que não muda depois de definida."
Em Python, convenção: nome em MAIÚSCULAS. Em JS, const. Em Java, final. Evita mudanças acidentais.
constfinalCAPS
MAX_USERS = 100
PI = 3.14159
# Convenção: MAIÚSCULAS
Se o valor nunca muda, deixe explícito. Facilita ler e refatorar.
EscopoT·11
"Escopo é onde uma variável existe e pode ser acessada."
Variável local só vive dentro da função; global vale em todo lugar. Bloco em JS tem escopo próprio (let/const).
localglobalblock
def f():
    x = 10     # local
f()
print(x)        # NameError
Globais facilitam — e bagunçam tudo. Prefira passar valores como argumento.
Type CastingT·12
"Converter um valor de um tipo para outro — int pra string, string pra float..."
Necessário quando recebe input (sempre string) ou ao combinar tipos diferentes. Pode falhar — pegue com try/except.
int()str()float()
x = input("idade: ")   # sempre str
age = int(x)            # cast pra int
print(age + 1)
Input é sempre texto. Converter pra número é o primeiro passo.

Bloco 3 · Operadores & Expressões

A Lógica da Matemática

Operadores são os verbos da programação — agem sobre valores e produzem resultados. Entender precedência e expressão é base de todo cálculo.
AritméticaComparaçãoLógicaAtribuiçãoPrecedência
10 termos · O·01 → O·10calcula
Operadores AritméticosO·01
"+ − × ÷ — matemática sobre números."
Os quatro básicos mais módulo % e potência **. Todo cálculo numérico começa aqui.
+-*/%
a, b = 10, 3
print(a + b, a - b, a * b)
print(a / b, a // b, a % b)
Divisão de int em Java trunca. Cast pra float quando quiser decimais.
Operadores de ComparaçãoO·02
"== != > < — comparam dois valores e produzem um booleano."
Base de toda decisão. == compara valor; em JS prefira ===; em Java pra strings use .equals().
==!=>=<=
age = 18
print(age >= 18)   # True
print("a" == "A")   # False
== compara valor; = atribui. Confundir é bug clássico.
Operadores LógicosO·03
"AND OR NOT — combinam booleanos pra produzir um booleano."
Python: and or not. JS/Java: && || !. Avaliação é curto-circuito: para na primeira certeza. Combina booleanos.
andornot&&
logged = True
admin = False
if logged and admin:
    print("painel")
Curto-circuito é seu aliado: user and user.name evita NPE.
Operadores de AtribuiçãoO·04
"= += -= *= /= — guardam um valor em uma variável."
O básico é =. Os compostos são atalhos: x += 1 é o mesmo que x = x + 1. Útil em incrementos.
=+=-=
total = 0
total += 10    # total = total + 10
total *= 2     # total = total * 2
Atalhos compostos deixam o código mais curto e claro.
Concatenação de StringsO·05
"Juntar pedaços de texto pra formar uma string maior."
Com + em quase todas as linguagens. Python tem f-strings, JS tem template literals — mais legíveis. Funciona em strings.
+f-stringformat()
name = "Cherry"
# Concatenação simples
print("Olá, " + name)
# f-string
print(f"Olá, {name}")
Concatenação em loop é lento. Pra muitas peças, use join().
MóduloO·06
"Operador % que dá o resto de uma divisão inteira."
Útil pra checar paridade (par/ímpar), ciclos, índices que dão volta. n % 2 == 0 = par. Base de muitos algoritmos com hash.
%restopar/ímpar
for i in range(10):
    if i % 2 == 0:
        print(i, "par")
% é a chave de muitos algoritmos: hashing, rotação, paridade.
Incremento / DecrementoO·07
"Somar ou subtrair 1 de uma variável — operação tão comum que ganhou atalho."
JS/Java: i++ e i--. Python não tem; usa i += 1. Importante em loops manuais.
++--+= 1
# Python
i = 0
i += 1           # i agora vale 1
# JS/Java equivalente: i++
i++ retorna antes; ++i retorna depois. Pré vs pós importa.
Precedência de OperadoresO·08
"A ordem em que operadores são avaliados em uma expressão."
Multiplicação antes da soma, lógicos depois de comparações. Use parênteses pra deixar claro — nunca confie só na regra.
()PEMDASordem
r = 2 + 3 * 4      # 14, não 20
r = (2 + 3) * 4    # 20
Na dúvida, parênteses. O leitor (e o você do futuro) agradecem.
Expressão vs ComandoO·09
"Expressão produz um valor. Comando faz alguma coisa."
2 + 3 é expressão (vale 5). print(x) é comando (faz, não retorna útil). Algumas linguagens borram a linha — como em ternários.
valuestatementexpr
# Expressão (produz valor)
total = 2 + 3
# Comando (faz ação)
print(total)
Expressões podem ser combinadas; comandos rodam isolados.
Efeitos ColateraisO·10
"Mudanças que uma função faz além de retornar — escrever em disco, alterar variável global, imprimir."
Função sem efeitos = pura (previsível). Função com efeitos = mais difícil de testar e debugar.
side effectpureglobal
contador = 0
def incr():
    global contador
    contador += 1   # efeito colateral
Funções com efeitos colaterais escondidos quebram testes. Minimize.

Bloco 4 · Controle de Fluxo

Decisões e Repetições

Sem if e loops, um programa só corre em linha reta. Aqui o código ganha capacidade de escolher e repetir — onde a lógica realmente começa.
IfElseWhileForBreakContinue
10 termos · C·01 → C·10decide e repete
IfC·01
"Executa um bloco só quando a condição é verdadeira."
O ponto de decisão mais simples. Sem if, seu programa não consegue reagir a nada. A condição vira booleano.
ifcondiçãobool
age = 20
if age >= 18:
    print("maior de idade")
If aninhado demais é cheiro de código. Considere extrair pra função.
Else / ElifC·02
"Caminhos alternativos quando o if é falso."
else = senão. elif (Python) ou else if = testa outra condição se a anterior falhou.
elseelifelse if
if score >= 90: nivel = "A"
elif score >= 70: nivel = "B"
else: nivel = "C"
Cadeia de elifs muito longa? Considere dict ou switch.
Switch / MatchC·03
"Ramificação múltipla pra um único valor — mais limpo que uma escada de elif."
JS/Java têm switch/case. Python 3.10+ tem match/case com pattern matching. Alternativa mais limpa que elif em cadeia.
switchcasematch
match day:
    case "sex": print("sextou")
    case "seg": print("foco")
    case _: print("normal")
Em JS, esquecer o break faz cair pro próximo case. Quase nunca é o que você quer.
WhileC·04
"Repete o bloco enquanto a condição for verdadeira."
Útil quando você não sabe quantas voltas vai dar. Atenção pra não criar loop infinito — use break se precisar.
whileloopcondição
i = 0
while i < 3:
    print(i)
    i += 1   # importante!
Esqueceu de mudar a variável da condição? Loop infinito. Confira sempre.
ForC·05
"Repete um número conhecido de vezes — controlado por contador ou coleção."
Em Python e JS modernos é for x in lista (veja For-Each). O for(i=0;i<n;i++) clássico ainda aparece em Java/JS.
forrangeiter
for i in range(5):
    print(i)   # 0..4
Sabe quantas iterações? for. Não sabe? while.
For-EachC·06
"Itera direto sobre cada item de uma coleção — sem se preocupar com índices."
Mais limpo e menos sujeito a bugs de off-by-one. for item in lista. Funciona em listas e dicts.
forEachiterin
frutas = ["maçã", "uva", "pera"]
for f in frutas:
    print(f)
Precisa do índice também? Use enumerate(lista) em Python.
BreakC·07
"Sai do loop imediatamente, antes do tempo previsto."
Útil quando achou o que procurava. Só sai do loop mais interno (em loops aninhados).
breakexitstop
for n in nums:
    if n == 42:
        print("achei!")
        break
Achou o que queria? break. Não force o loop a terminar à toa.
ContinueC·08
"Pula pro próximo ciclo do loop, ignorando o resto da iteração atual."
Bom pra filtrar valores sem criar um if que engloba todo o bloco.
continueskipnext
for n in range(10):
    if n % 2 == 0:
        continue   # pula pares
    print(n)
continue deixa o caminho feliz menos indentado. Use com moderação.
Operador TernárioC·09
"If-else em uma linha — retorna um de dois valores baseado em uma condição."
Em Python: a if cond else b. Em JS/Java: cond ? a : b. Ótimo pra atribuições simples.
? :if-else expr
tipo = "adulto" if age >= 18 else "menor"
# JS: const tipo = age >= 18 ? "adulto" : "menor"
Ternário aninhado fica ilegível. Quando passar disso, volte pro if/else.
Loops AninhadosC·10
"Loop dentro de outro loop — necessário pra matrizes e combinações."
Cuidado: complexidade (veja Big O) explode. Dois loops de N viram N². Três viram N³.
nestedmatrixO(n²)
for i in range(3):
    for j in range(3):
        print(i, j)
Três ou mais loops aninhados? Repensar é quase sempre melhor que continuar.

Bloco 5 · Funções

A Arte da Reutilização

Função é a unidade básica de organização de código. Bem feita, transforma 100 linhas confusas em 10 linhas claras.
defreturnparâmetrospurarecursãolambda
12 termos · N·01 → N·12compõe
FunçãoN·01
"Bloco de código com nome que pode ser chamado várias vezes."
Recebe entrada (parâmetros), faz algo, retorna saída. Unidade de reuso e abstração.
deffunctionreturn
def saudacao(name):
    return f"Olá, {name}"
print(saudacao("Cherry"))
Função boa faz uma coisa, faz bem feito. Se faz duas, vire duas funções.
ParâmetroN·02
"Variável de entrada listada na definição da função."
É o "espaço reservado" pro valor que vai chegar. Não confunda com argumento (que é o valor real passado).
paraminputsignature
# a, b são parâmetros
def soma(a, b):
    return a + b
Mais de 3-4 parâmetros? Provavelmente está pedindo demais. Considere agrupar.
ArgumentoN·03
"O valor concreto passado pra função na hora de chamar."
Parâmetro está na assinatura; argumento está na chamada. Em Python, pode passar por posição ou nome.
argcallkwarg
def soma(a, b): return a + b
# 3 e 4 são argumentos
soma(3, 4)
soma(a=3, b=4)   # nomeados
Nomeados (kwargs) deixam a chamada autoexplicativa — vale o tempo extra de digitar.
Valor de RetornoN·04
"O valor que a função devolve pra quem a chamou."
Sem return, Python retorna None. Você pode usar o retorno em expressões ou guardar em variável.
returnoutputresult
def dobro(x): return x * 2

resultado = dobro(5)   # 10
Retornar cedo (early return) deixa o código menos indentado e mais claro.
Função PuraN·05
"Mesma entrada sempre dá a mesma saída, sem efeitos colaterais."
Não depende de estado externo, não muda nada fora (sem efeitos colaterais). Testável, previsível, fácil de paralelizar.
purestatelessdeterministic
# Pura
def soma(a, b): return a + b
# Impura (depende do tempo)
def agora(): return datetime.now()
Função pura é fácil de testar. Empurre estado pras bordas do programa.
RecursãoN·06
"Função que chama ela mesma — com um caso base pra parar."
Elegante pra problemas em árvore ou divisão. Sem caso base = stack overflow (estoura a pilha).
recursivebase casestack
def fact(n):
    if n <= 1: return 1   # base
    return n * fact(n - 1)
Recursão é loop em outra roupa. Para entender, desenhe a pilha.
ClosureN·07
"Função que lembra das variáveis do escopo onde foi criada."
Mesmo depois daquele escopo sumir, a função interna ainda acessa. Base de muitos padrões em JS.
closurecapturescope
def make_counter():
    count = 0
    def inc():
        nonlocal count
        count += 1; return count
    return inc
Closure cria estado privado sem precisar de classe.
Lambda / ArrowN·08
"Função anônima, geralmente curta, usada inline como argumento."
Python: lambda x: x*2. JS: x => x*2. Ótima pra callbacks e map/filter — base de HOFs.
lambda=>anônima
nums = [1, 2, 3]
dobros = list(map(lambda x: x*2, nums))
print(dobros)   # [2,4,6]
Lambda de 1-2 linhas é elegante. Maior que isso, dê nome com def.
Argumentos PadrãoN·09
"Valor pré-definido pra um parâmetro quando o chamador não passa nada."
Deixa funções mais flexíveis sem precisar sobrecarregar. Em Python, evite mutáveis (list/dict) como default.
default=optional
def saudar(name, msg="Olá"):
    return f"{msg}, {name}"

saudar("Cherry")         # Olá, Cherry
Cuidado: def f(x=[]) compartilha a mesma lista entre chamadas. Use None.
*args / **kwargsN·10
"Argumentos de número variável — *args coleta posicionais, **kwargs coleta nomeados."
Quando você não sabe quantos argumentos vão chegar. Comum em decoradores e wrappers.
*args**kwargsvarargs
def log(*args, **kwargs):
    print(args)    # tupla
    print(kwargs)  # dict
log(1, 2, name="Cherry")
Use com cuidado: assinaturas vagas viram caixas-pretas.
Escopo de FunçãoN·11
"Variáveis declaradas dentro da função só existem dentro dela."
Quando a função termina, suas variáveis somem. Isso é bom — evita poluição global e bugs estranhos.
localstack frame
def f():
    x = 10     # local
f()
print(x)        # NameError
Cada chamada cria um escopo novo. Funções não compartilham variáveis locais.
Higher-Order FunctionsN·12
"Funções que recebem ou retornam outras funções."
Base do paradigma funcional. map, filter, reduce, decoradores — tudo isso é HOF. Use com lambdas.
mapfilterdecorator
nums = [1, 2, 3, 4]
pares = list(filter(lambda x: x%2==0, nums))
HOFs encurtam código. map+filter+reduce substituem muito loop manual.

Bloco 6 · Estruturas de Dados

As Formas dos Dados

Escolher a estrutura certa é o que separa um algoritmo rápido de um lento. Cada uma tem força e fraqueza — saber onde aplicar é metade do jogo.
ArrayListaDictSetStackQueueTreeGraph
12 termos · S·01 → S·12estrutura
ArrayS·01
"Sequência de valores de tamanho fixo, acessada por índice."
Em memória contígua: acesso por índice é O(1). Em Java, tamanho é fixo na criação.
arrayindexO(1)
// Java
int[] nums = {1, 2, 3};
nums[0] = 10;       // acesso O(1)
Acesso por índice é instantâneo. Inserir no meio, não.
ListaS·02
"Array dinâmico — cresce e encolhe conforme você adiciona ou remove."
Python list, JS Array, Java ArrayList. Acesso O(1), inserção no fim ~O(1), no meio O(n).
listappenddynamic
nums = [1, 2, 3]
nums.append(4)       # cresce
nums.pop(0)         # remove primeiro
Inserir/remover no fim é rápido. No começo ou meio, todos os outros se deslocam.
TuplaS·03
"Sequência imutável e ordenada — não pode mudar depois de criada."
Útil pra valores que sempre andam juntos (coordenadas, dimensões). Pode ser chave de dict; lista não pode.
tupleimmutable()
ponto = (3, 4)        # tupla
x, y = ponto         # desempacotamento
# ponto[0] = 10   # ERRO: imutável
Imutabilidade evita bugs sutis. Tupla é segura pra compartilhar entre threads.
Dicionário / MapS·04
"Pares chave-valor com busca rápida pela chave."
Lookup em O(1) médio (usa hashing). Ideal pra contagem, cache, modelagem de entidades.
dictmaphash
user = {"id": 1, "name": "Cherry"}
print(user["name"])    # O(1)
user["age"] = 25          # adiciona
Sempre que pensar "buscar por nome rápido", pense em dict.
SetS·05
"Coleção desordenada de itens únicos — sem duplicatas."
Verificar pertencimento é O(1) (usa hash). Suporta união, interseção, diferença.
setuniquein
visitas = {"a", "b", "a"}   # {"a","b"}
print("a" in visitas)        # True
visitas.add("c")
Remover duplicatas de lista: list(set(lista)). Truque clássico.
Pilha (Stack)S·06
"LIFO — Last In, First Out. O último que entra é o primeiro que sai."
Como pilha de pratos. Operações: push (empilha), pop (desempilha). Usada em chamadas de função, undo, parsing. Base da recursão.
LIFOpushpop
stack = []
stack.append(1)        # push
stack.append(2)
print(stack.pop())   # 2
Toda chamada de função empilha na call stack. Stack overflow é ela transbordando.
Fila (Queue)S·07
"FIFO — First In, First Out. O primeiro que entra é o primeiro que sai."
Como fila de banco. Usada em BFS (veja grafo), processamento assíncrono, mensageria.
FIFOenqueuedequeue
from collections import deque
q = deque()
q.append("a")          # enqueue
print(q.popleft())   # "a"
Use deque, não list.pop(0) — é O(n).
Lista EncadeadaS·08
"Sequência de nós onde cada um aponta pro próximo."
Inserir/remover no meio é O(1) (já tendo o nó). Acesso por índice é O(n). Singly ou doubly linked.
nodenextpointer
class Node:
    def __init__(self, val):
        self.val = val
        self.next = None
Em prática você raramente implementa uma do zero — mas precisa entender pra entender árvores e grafos.
ÁrvoreS·09
"Estrutura hierárquica com raiz e filhos — sem ciclos."
Modela hierarquias: filesystem, DOM, organograma. Binary tree, BST, heap, trie — variantes pra cada problema. Funcionam bem com recursão.
treerootleafBST
class TreeNode:
    def __init__(self, val):
        self.val = val
        self.left = self.right = None
Recursão e árvore foram feitas uma pra outra. Boa parte dos algoritmos sai natural.
GrafoS·10
"Nós conectados por arestas — relações arbitrárias entre coisas."
Redes sociais, mapas, dependências. Pode ter ciclos (diferente de árvore). BFS e DFS são as travessias clássicas.
graphedgeBFSDFS
# Adjacency list
grafo = {
    "A": ["B", "C"],
    "B": ["A", "D"],
}
Se você consegue desenhar como bolinhas e linhas, é um grafo.
Hash TableS·11
"Estrutura que mapeia chave em slot usando uma função de hash."
Base de dict, set e Map. Lookup médio O(1). Colisões resolvidas por chaining ou open addressing.
hashbucketcollision
# dict é uma hash table
d = {"a": 1, "b": 2}
print(d["a"])   # O(1) médio
Hash table é o canivete suíço. Toda vez que pensar "preciso buscar rápido", pense nela.
Big OS·12
"Notação que descreve como um algoritmo escala com o tamanho da entrada."
O(1) constante · O(log n) logarítmico · O(n) linear · O(n²) quadrático (típico de loops aninhados) · O(2ⁿ) exponencial.
O(1)O(n)O(n²)
# O(1): acesso em array
arr[5]
# O(n): loop simples
for x in arr: ...
# O(n²): loop aninhado
Em entradas grandes, O(n²) é tragédia. Big O paga estudo.

Bloco 7 · Programação Orientada a Objetos

Código como Objetos

POO modela o mundo em coisas que têm estado e comportamento. Funcionalidade não é tudo — organização é o que faz código de produção sobreviver anos.
ClasseObjetoHerançaPolimorfismoEncapsulamento
12 termos · P·01 → P·12modela
ClasseP·01
"Classe é o molde — descreve a forma e o comportamento de objetos."
Define quais atributos e métodos cada instância vai ter. Sem instâncias, é só uma planta.
classblueprinttype
class User:
    def __init__(self, name):
        self.name = name
Classe é receita; objeto é o bolo. Sempre são duas coisas distintas.
ObjetoP·02
"Objeto é um pacote concreto de estado + comportamento, criado a partir de uma classe."
Cada objeto tem seus próprios valores de atributos. Em Python, quase tudo é objeto. Criado pelo construtor.
objectnewinstance
u = User("Cherry")   # cria objeto
print(u.name)        # "Cherry"
Dois objetos da mesma classe são independentes. Mudar um não afeta o outro.
InstânciaP·03
"Instância é um objeto específico criado a partir de uma classe."
Termo praticamente sinônimo de objeto, mas enfatiza a relação com a classe que o gerou.
instanceisinstance
a = User("A")
b = User("B")
isinstance(a, User)   # True
"Uma instância de User" = um objeto User. As palavras dançam juntas.
AtributoP·04
"Variável que pertence a um objeto — guarda seu estado."
Acessada com obj.attr. Pode ser de instância (única por objeto) ou de classe (compartilhada).
attrfieldstate
class User:
    species = "human"   # de classe
    def __init__(self, name):
        self.name = name   # de instância
Atributo é dado; método é função. Juntos formam o objeto.
MétodoP·05
"Função ligada a uma classe — opera sobre os atributos de um objeto."
Recebe self (Python) ou this (JS/Java) implicitamente, referenciando o objeto que chamou.
methodselfthis
class User:
    def greet(self):
        return f"Olá, {self.name}"
Método opera no objeto. Função solta opera no que você passa.
ConstrutorP·06
"Método especial chamado automaticamente na criação do objeto."
Python: __init__. JS: constructor. Java: método com nome da classe. Inicializa atributos.
__init__constructornew
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
p = Point(3, 4)
O construtor garante que todo objeto nasce em estado válido.
HerançaP·07
"Uma classe estende outra, reaproveitando atributos e métodos."
Subclasse herda da superclasse. Modela "é um(a)": Cachorro é um Animal. Use com moderação — composição costuma ser melhor. Habilita polimorfismo.
extendssuperparent
class Animal:
    def speak(self): return "..."
class Dog(Animal):
    def speak(self): return "woof"
Hierarquia profunda vira pesadelo. Prefira composição quando não for "é-um" claro.
PolimorfismoP·08
"Mesma interface, comportamento diferente por classe."
Chama obj.speak() sem saber se é Dog, Cat ou Cow — cada um responde do seu jeito. Acontece via herança ou interface.
overrideduck typing
for a in [Dog(), Cat()]:
    print(a.speak())   # woof, meow
Polimorfismo deixa código aberto a extensão sem precisar mudá-lo.
EncapsulamentoP·09
"Esconder o estado interno por trás de uma interface controlada."
Quem usa só chama métodos públicos; o estado é privado. Em Python, convenção: _attr = privado.
privatepublicgetter
class Account:
    def __init__(self):
        self._balance = 0   # privado
    def deposit(self, v):
        self._balance += v
Esconder estado libera você pra mudar implementação sem quebrar usuários.
InterfaceP·10
"Contrato listando métodos que uma classe se compromete a implementar."
Java tem interface. Python tem ABCs e duck typing. Define "como conversar", não "como funciona" — pura abstração.
interfacecontractabstract
// Java
interface Drawable {
    void draw();
}
Programe pra interface, não pra implementação. Princípio que envelhece bem.
AbstraçãoP·11
"Focar no essencial e esconder a complexidade do resto."
Você dirige sem saber como o motor funciona. Boas abstrações vendem ideias claras sem revelar detalhes — princípio central da interface.
abstractlayer
# Abstração: você não precisa saber
# como sorted funciona internamente
sorted([3, 1, 2])     # [1, 2, 3]
Abstração ruim vaza. Boa abstração some — você nem percebe que existe.
this / selfP·12
"Referência ao objeto atual dentro de um método."
Python passa self explícito; JS/Java usam this implícito. Permite método acessar e modificar o próprio objeto.
selfthisinstance
class Counter:
    def __init__(self):
        self.value = 0
    def inc(self):
        self.value += 1
Esquecer self em Python é um dos erros mais comuns de iniciante.

Bloco 8 · Erros & Debug

Quando as Coisas Quebram

Errar é parte do trabalho. Saber tratar, investigar e prever falhas separa código de brincadeira de código de produção.
ExceptionTryStack TraceLinterEdge Case
10 termos · E·01 → E·10investiga
ExceçãoE·01
"Erro inesperado que interrompe a execução normal do programa."
Divisão por zero, arquivo inexistente, lista vazia (clássico edge case). Se não for capturada, o programa morre.
exceptionerrorraise
int("abc")
# ValueError: invalid literal for int()
Exceção é informação, não fracasso. Ela conta o que foi inesperado.
Try / CatchE·02
"Bloco pra tentar uma operação e reagir se der erro."
Python: try/except. JS/Java: try/catch. Sempre capture exceções específicas, nunca tudo.
tryexceptfinally
try:
    age = int(input())
except ValueError:
    print("número inválido")
except Exception nu engole bugs. Capture só o que você espera.
Throw / RaiseE·03
"Disparar uma exceção manualmente do seu código."
Útil pra sinalizar erro de uso, validação falha, condição impossível. Não silencie — fale alto e claro. Capture com try/catch.
raisethrow
def div(a, b):
    if b == 0:
        raise ValueError("b não pode ser 0")
    return a / b
Falha cedo e barulhento é melhor que dado errado silencioso.
Stack TraceE·04
"Lista de funções que estavam ativas no momento do erro."
Leia de baixo pra cima — a última linha é geralmente onde o erro ocorreu. Pista mais valiosa em qualquer bug. Cada entrada vem da call stack.
tracebackcallstack
File "app.py", line 10
  total = soma(a, b)
File "app.py", line 5
  return a + b
TypeError: ...
Não ignore stack trace — leia. Ela conta exatamente onde e como o problema apareceu.
BreakpointE·05
"Marca em uma linha que pausa a execução pra você inspecionar."
Coloque na IDE ou via breakpoint() em Python 3.7+. Inspecione variáveis no debugger.
breaksteppause
def calc(x):
    breakpoint()   # pausa aqui
    return x * 2
Breakpoint vence print quando você não sabe onde o estado virou estranho.
print / console.logE·06
"Debug rápido por inspeção visual — imprime valor pra ver o que está acontecendo."
Não é elegante, mas funciona. Aprenda a usar e a remover antes de commit (veja controle de versão).
printlogdebug
def somar(a, b):
    print(f"a={a}, b={b}")
    return a + b
Print debugging não é vergonha. Funciona. Só não esqueça de limpar depois.
LinterE·07
"Ferramenta que analisa o código estaticamente e aponta erros e cheiros."
ESLint, Pylint, Ruff, Flake8. Pega bugs antes de rodar e padroniza estilo. Integre na IDE.
eslintpylintruff
# Roda linter na linha de comando
ruff check .
eslint src/
Linter é seu primeiro revisor de código. Use sempre, mesmo em projeto solo.
DebuggerE·08
"Ferramenta interativa pra andar pelo código linha por linha em execução."
Pause em um breakpoint, inspecione variáveis, mude valores, pule. Toda IDE séria tem um. Aprender economiza horas.
stepwatchpdb
# Python
import pdb; pdb.set_trace()
# Comandos: n (next), s (step), c (cont)
Print mostra o passado; debugger mostra o presente. Ele é mais poderoso.
Edge CaseE·09
"Entrada incomum que quebra código que funciona pro caso normal."
Lista vazia, número negativo, string com emoji, divisão por zero, valor null. Onde os bugs adoram morar.
corner caseboundary
def media(lista):
    return sum(lista) / len(lista)
media([])   # ZeroDivisionError!
"E se for vazio? E se for negativo? E se for null?" — repita até esgotar.
Erro de Sintaxe vs RuntimeE·10
"Dois momentos diferentes em que erros aparecem."
Sintaxe: parser pegou antes de rodar. Runtime: só apareceu durante a execução. Cada um se previne diferente.
parseruntimeSyntaxError
# Sintaxe (programa nem inicia)
if x > 0
    print(x)        # falta :

# Runtime (só ao rodar)
int("abc")         # ValueError
Sintaxe morre cedo (bom). Runtime morre tarde — testes existem pra antecipar.

Bloco 9 · Boas Práticas

Código que Dura

O que separa profissional de amador não é só fazer funcionar — é fazer durar. Estes hábitos transformam código descartável em código que sobrevive a você.
NamingDRYSOLIDRefactorTestesGit
10 termos · B·01 → B·10com cuidado
Convenções de NomesB·01
"Escolha nomes que se explicam sozinhos — ninguém deveria precisar de comentário pra entender."
snake_case (Python), camelCase (JS/Java), PascalCase (classes), UPPER (constantes). Siga a convenção da linguagem.
snake_casecamelCase
# Ruim
n = getN(d)
# Bom
user_count = get_user_count(database)
Tempo gasto escolhendo nome bom é tempo poupado lendo código depois.
IndentaçãoB·02
"Estrutura visual do código — mostra o que pertence a quem."
Em Python, é obrigatória (parte da sintaxe). Em outras, é convenção forte. Use sempre 2 ou 4 espaços consistentes.
indent2 spaces4 spaces
def f(x):
    if x > 0:
        print("pos")
    else:
        print("não-pos")
Indentação inconsistente é hostil ao leitor. Use formatter (Prettier, Black, gofmt).
ComentáriosB·03
"Explique o PORQUÊ, não o O QUÊ — o código já mostra o quê."
Bom comentário explica decisão, contexto, gotcha. Ruim só repete o nome da função. Código autoexplicativo é melhor (com bom naming).
#//why
# Ruim: redundante
x += 1   # incrementa x

# Bom: explica decisão
retry_count = 3   # API instável, 3 evita falsos negativos
Comentário só dura se for útil. Se for óbvio, apague. Nome bom > comentário.
DRYB·04
"Don't Repeat Yourself — cada fato vive em um único lugar autoritativo."
Lógica duplicada é dívida que cresce. Mudou em um lugar, esqueceu no outro = bug. Resolva extraindo uma função.
DRYextractrefactor
# Ruim: duplicado
print(price * 1.1)
print(total * 1.1)
# Bom: regra em um lugar
def with_tax(v): return v * 1.1
DRY é regra, não dogma. Repetição superficial às vezes é mais clara que abstração ruim.
SOLIDB·05
"Cinco princípios de design pra POO sustentável."
S Single Responsibility · O Open/Closed · L Liskov · I Interface Segregation · D Dependency Inversion. Aplica-se a classes.
SRPOCPLSP
# SRP: cada classe, uma razão pra mudar
class User: ...           # dados
class UserRepository: ... # persiste
class UserEmailer: ...    # envia email
SOLID é guia, não bíblia. Aprenda os princípios, mas aplique com julgamento.
RefatorarB·06
"Melhorar a estrutura do código sem mudar o que ele faz."
Renomear, extrair função, simplificar condição. Sempre com testes pra garantir que nada quebrou.
refactorcleanuprename
# Antes
if u and u.age > 18 and u.active:
# Depois (extraído)
if can_purchase(u):
Refatore quando entender — não enquanto luta pra entender. Refatorar no escuro vira ruína.
Code ReviewB·07
"Colegas leem o código uns dos outros antes de virar produção."
Pega bugs, espalha conhecimento, mantém padrão. Não é julgamento — é trabalho em equipe. Geralmente via Pull Request.
PRreviewfeedback
# Fluxo típico
git checkout -b feat/login
# ... commits ...
git push origin feat/login
# Abre Pull Request
Bom reviewer comenta o porquê, não só o "muda isso". Aprenda com o feedback.
Controle de VersãoB·08
"Sistema que registra cada mudança no código — Git é o padrão."
Commits, branches, merges, history. Permite voltar atrás, colaborar e experimentar sem medo. Fluxo padrão para code review.
gitcommitbranch
git status
git add .
git commit -m "add login"
git push
Commits pequenos e frequentes contam uma história. Commits grandes contam confusão.
TestesB·09
"Verificações automatizadas que confirmam que o código faz o que deveria."
Unit (função isolada), integration (peças juntas), e2e (fluxo completo). Testes são rede de proteção pra refatorar.
unitpytestassert
def test_soma():
    assert soma(2, 3) == 5
    assert soma(-1, 1) == 0
Sem testes, refatorar é roleta-russa. Com testes, você muda código com confiança.
DocumentaçãoB·10
"Escrever pra quem vai ler depois — inclusive você daqui a seis meses."
README, docstrings, ADRs, exemplos de uso. Documentação ruim é pior que nenhuma; mantenha viva — versionada com Git.
READMEdocstringdocs
def calc_iva(price):
    """Aplica 21% de IVA ao preço.

    Args:
        price: valor em euros, sem IVA.
    Returns:
        valor com IVA.
    """
    return price * 1.21
