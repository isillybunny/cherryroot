Hardware
Everything you can physically touch on a computer.

① Simple Definition
"Everything you can physically touch on a computer."
Hardware is all the physical components — input devices (keyboard, mouse), output devices (monitor, speakers), storage (HDD, SSD), processing (CPU) and memory (RAM). Software cannot exist without hardware to run on.
② Explanation

hardware receiving it and ends with hardware displaying the result. The motherboard connects all components. In security, physical access is often the most dangerous — an attacker with physical access can bypass most software protections.

③ Keywords
hardwareinput deviceoutput devicestorageSSDHDDmotherboardperipheral
④ Where It Appears
Every physical device
Computers and laptops
Servers
Phones and IoT
⑤ How It Works
1Input device receives your action
2Signal travels through the motherboard
3CPU processes the instruction
4Result sent to output device
⑥ Attack & Defense
Attack
  • Physical install of keyloggers
  • Stealing drives to extract data
Defense
  • Physical security controls
  • Full-disk encryption
  • Locked BIOS
⑦ Remember This
Software is what you see. Hardware is what makes it possible. No hardware = no computer.

CPU
The brain of the computer — executes every single instruction.
B·02pc basicsprocessing
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"The brain of the computer — it executes every instruction."
The CPU processes everything. Every calculation, every program, every action goes through it. It works directly with RAM to fetch and run instructions.
② Explanation

CPUs have cores — more cores means more things run simultaneously. The clock speed is in GHz — billions of cycles per second. In security, exploits like buffer overflows are designed around how the CPU handles memory. CPU-level bugs like Spectre can leak data across isolated processes.

③ Keywords
CPUprocessorcorex86x64ARMregistersclock speed
⑤ How It Works
1Fetches instruction from RAM
2Decodes the instruction
3Executes the operation
4Stores result back to RAM
5Repeats billions of times per second
⑥ Attack & Defense
Attack
  • Spectre and Meltdown CPU data leaks
  • Buffer overflows target instruction flow
Defense
  • Keep firmware and OS updated
  • CPU patches applied at OS level
⑦ Remember This
The CPU reads instructions from memory. This is why memory exploits work — they put malicious instructions where the CPU will find them.

cherry root › block 0 › B·03
RAM
Fast temporary memory — holds everything currently active.
B·03pc basicsmemory
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Fast temporary memory the computer uses while running programs."
RAM holds everything currently active. Wiped completely when you turn the computer off. Much faster than disk but nothing persists.
② Explanation

When you open a file it loads from disk into RAM so the CPU can access it quickly. RAM is split into the stack (function calls, local variables) and heap (dynamic data). In security, RAM can hold passwords and encryption keys — making RAM forensics a real attack and investigation technique.

③ Keywords
RAMvolatile memoryheapstackmemory dumpbufferRAM forensics
⑤ How It Works
1You open a program
2OS loads it from disk into RAM
3CPU reads and writes to RAM as it runs
4When closed, RAM space is freed
5On shutdown, all RAM is wiped
⑥ Attack & Defense
Attack
  • RAM dumps extract passwords from memory
  • Cold boot attacks recover RAM after shutdown
Defense
  • Full-disk encryption
  • Lock screen when away
⑦ Remember This
RAM = volatile (disappears on shutdown). Disk = persistent (stays). Critical difference in forensics.

Software
Instructions that tell hardware what to do — no physical form.
B·04pc basicsprograms
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Instructions that tell hardware what to do."
Software is any program, app or script. It exists only as code and runs on hardware using the CPU and RAM. No physical form.
② Explanation

Divided into system software (OS), application software (browser, apps) and malicious software (malware). Every app and website runs as software. The difference between a program and malware is only intent — the mechanics are the same.

③ Keywords
softwareapplicationsource codeexecutablebinaryfirmware
⑤ How It Works
1Developer writes source code
2Compiled into binary the CPU understands
3OS loads the binary into RAM
4CPU executes line by line
⑥ Attack & Defense
Attack
  • Malware disguises as legit software
  • Trojanized apps carry hidden payloads
Defense
  • Only install from trusted sources
  • Keep everything updated
⑦ Remember This
The line between good software and malware is intent. The mechanics are exactly the same.

Network
The fundamental concept that everything else builds on.
N·01networkfundamentals
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Two or more devices that can talk to each other form a network."
A network is any group of connected devices that share data. Two laptops on the same WiFi form a network. The internet is a network of networks.
② Explanation

Every device on a network has an identity (an IP address) and uses agreed-upon rules (protocols) to communicate. Without a network, computers are isolated islands. With one, they can share files, services, and attacks.

Networks come in different sizes: a single home WiFi (LAN), a corporate network spanning buildings (WAN), or the entire internet. The principles of how they work are the same — only the scale changes.

③ Keywords
networkdeviceconnectioncommunicationLANWANtopology
④ Where It Appears
Every connected device — phone, laptop, IoT
Office and home WiFi
Cloud services and data centers
Every cyberattack starts on a network
⑤ How It Works

  • Network segmentation — divide into isolated zones
  • Monitor traffic with IDS/IPS for anomalies

LAN vs WAN
Local versus wide — two different security models.
N·02LANWAN
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"LAN = your house. WAN = the city. Internet = the world."
A LAN (Local Area Network) covers a limited area like a home, office, or building. A WAN (Wide Area Network) connects multiple LANs across cities, countries, or the entire planet — the internet itself is the largest WAN.
② Explanation

Inside a LAN, devices typically trust each other and traffic flows freely. The boundary between LAN and WAN is where firewalls and routers sit, deciding what gets in and out.

This trust difference matters. Many internal services have weak authentication because they assume "you are already inside, so you are trusted." This is why getting onto a corporate LAN — even briefly — is so valuable to an attacker.

③ Keywords
LANWANsubnetrouterfirewallprivate networkpublic network
④ Where It Appears
Home WiFi = LAN
Office network = LAN connected to WAN
Internet = the global WAN
Cellular networks bridge devices to the WAN
⑤ How It Works
Devices on a LAN typically share a private IP range (192.168.x.x or 10.x.x.x) and communicate directly via switches. To reach the WAN, traffic goes through a router that translates private IPs to a public one (NAT). Firewalls at the LAN/WAN boundary filter what is allowed.
⑥ Attack & Defense
Attack
  • Once inside the LAN, attacker has visibility to many internal services
  • WAN-exposed services are first scan targets for any external attacker
  • Pivoting — using a compromised LAN device to attack other LAN devices
Defense
  • Zero trust internally — never assume LAN traffic is safe
  • Strong firewalls between LAN and WAN
  • Microsegmentation — divide LAN into smaller trust zones
⑦ Remember This
The LAN/WAN boundary is critical, but assuming everything inside the LAN is safe is the most common mistake. Treat internal traffic with skepticism too.

OSI Model
Seven layers that explain how data travels — every attack lives somewhere in this stack.
N·03OSIlayers
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Seven layers stacked: Physical, Data Link, Network, Transport, Session, Presentation, Application."
The OSI model is a conceptual framework that breaks network communication into 7 layers. Each layer has a specific job — from physical cables (L1) to applications you click on (L7). Knowing layers helps locate where attacks happen.
② Explanation

The mnemonic: "Please Do Not Throw Sausage Pizza Away" (Physical, Data Link, Network, Transport, Session, Presentation, Application).

Each layer talks to the layer above and below. When you send data, it travels DOWN the layers (encapsulated with headers), travels across the network, then travels UP the layers on the other side (de-encapsulated). Different attacks target different layers — knowing which layer something happens at tells you what defenses apply.

③ Keywords
layer 1layer 2layer 3layer 4layer 7encapsulationTCP/IP
④ Where It Appears
Network engineering documentation
Cybersecurity training and certifications
Wireshark protocol analysis
Discussing where attacks happen ("a layer 7 attack")
⑤ How It Works
Layer 1 (Physical) — cables, radio waves, electrical signals. Layer 2 (Data Link) — MAC addresses, switches, Ethernet frames. Layer 3 (Network) — IP addresses, routers, packets. Layer 4 (Transport) — TCP/UDP, ports, segments. Layers 5-7 (Session, Presentation, Application) — sessions, encryption, app protocols like HTTP. Data flows down the stack on send, up on receive.
⑥ Attack & Defense
Attack
  • L2 ARP spoofing — poison the local network
  • L3 IP spoofing — fake source IP
  • L4 SYN floods — exhaust connection tables
  • L7 SQL injection, XSS — application-layer attacks
Defense
  • Defense in depth — protect each layer separately
  • L2 — port security, dynamic ARP inspection
  • L3-L4 — firewalls and rate limiting
  • L7 — WAF, input validation, secure coding
⑦ Remember This
Layers are not just theory. When something breaks or gets attacked, ask "which layer?" — that determines what tool you reach for.

TCP
Reliable, ordered delivery — the workhorse of the internet.
N·04TCPtransport
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"TCP is the protocol that guarantees your data arrives, in order, with errors detected."
TCP (Transmission Control Protocol) is a connection-based protocol that ensures data arrives reliably and in the correct order. Used by web browsing, email, SSH, and most things where missing data would be a problem.
② Explanation

TCP starts with a three-way handshake: SYN (client says hi), SYN-ACK (server replies), ACK (client confirms). After this, both sides know the connection is established and can exchange data with confidence.

TCP numbers every byte sent. If something is lost, it gets retransmitted. If packets arrive out of order, TCP reorders them. This reliability has a cost — TCP is slower than UDP because of the bookkeeping. But for things that matter (like loading a webpage), reliability beats speed.

③ Keywords
TCPthree-way handshakeSYNSYN-ACKACKreliableconnection-oriented
④ Where It Appears
HTTP/HTTPS websites
SSH remote access
SMTP/IMAP email
FTP file transfer
⑤ How It Works
Connection setup uses the SYN/SYN-ACK/ACK handshake. Once established, data is broken into segments numbered sequentially. Each segment requires acknowledgment. Lost segments are retransmitted. Connection ends with FIN packets from both sides.
⑥ Attack & Defense
Attack
  • SYN flood — send many SYN without completing handshake, exhausting connection slots
  • TCP RST injection — forge reset packets to drop legitimate connections
  • Sequence number prediction (mostly historical, but watch on weak implementations)
Defense
  • SYN cookies — handle SYN floods without keeping state
  • Connection rate limiting per source IP
  • Strong randomization of sequence numbers
⑦ Remember This
TCP equals reliable. Use it when missing data would break your application — websites, file transfers, logins.

UDP
Fast, lightweight, no guarantees.
N·05UDPtransport
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"UDP fires and forgets — fast but unreliable."
UDP (User Datagram Protocol) is connectionless and stateless. It just sends packets without confirmation. Used when speed matters more than reliability — gaming, video streaming, DNS lookups, VoIP.
② Explanation

UDP has no handshake, no retransmission, no ordering. You send a packet and hope it arrives. The trade-off is much lower overhead — perfect for things where late data is worse than missing data.

For example, in a live video call, retransmitting a lost frame would just create lag. Better to skip it and keep going. Same with online games — last second old position doesn't matter, current position does. UDP's simplicity also makes it attractive for amplification attacks.

③ Keywords
UDPconnectionlessstatelessdatagramfastunreliable
④ Where It Appears
DNS queries (port 53)
VoIP calls
Online gaming
Video streaming
Network discovery protocols
⑤ How It Works
No connection setup. Each packet (datagram) is independent. Sender sends, receiver receives — or does not. No acknowledgments, no retransmission. Application layer handles any reliability needed (or accepts loss).
⑥ Attack & Defense
Attack
  • UDP amplification — small request, huge response (DNS, NTP, memcached) directed at victim
  • Spoofing source IP is trivial since there is no handshake to verify
  • UDP flood — overwhelm target with junk packets
Defense
  • Rate limiting on UDP-based services
  • Anti-spoofing filters at the network edge (BCP 38)
  • Disable open DNS resolvers and amplification vectors
⑦ Remember This
UDP equals fast and risky. Great for real-time, terrible if reliability matters. Most amplification attacks use UDP because it cannot be verified.

MAC Address
The physical address burned into every network card.
N·06MAClayer 2
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Your network card has a unique MAC. Routers see IPs, switches see MACs."
A MAC address is a 48-bit identifier assigned to a network interface card (NIC). Format: 6 hex pairs like AA:BB:CC:DD:EE:FF. It is the physical address used at Layer 2 — within a local network, devices find each other by MAC.
② Explanation

MAC addresses are theoretically unique and assigned by manufacturers (the first 3 octets identify the vendor). However, almost any modern OS lets you change your MAC — useful for privacy on public WiFi, also useful for attackers.

MAC addresses only matter within a local network segment. Once traffic crosses a router, MACs are stripped and replaced. So a MAC tells you who is on the local network — never anything about distant attackers.

③ Keywords
MAC addressNICOUIlayer 2physical addressARP
④ Where It Appears
Network admin tools (ifconfig, ip link, ipconfig)
WiFi access logs
DHCP server records
Network access control (NAC) systems
⑤ How It Works
Each network packet on a LAN has source MAC, destination MAC, source IP, destination IP. Switches forward based on MAC tables they build by watching traffic. ARP (Address Resolution Protocol) maps IPs to MACs so the network knows where to deliver.
⑥ Attack & Defense
Attack
  • MAC spoofing — change your MAC to bypass MAC-based filtering or impersonate another device
  • ARP spoofing — claim to be the gateway, intercepting all traffic
  • MAC flooding — fill the switch CAM table to force it into hub mode (broadcasts everything)
Defense
  • Port security on switches — limit MACs per port
  • Dynamic ARP Inspection (DAI) — block invalid ARP responses
  • 802.1X authentication — verify identity before granting network access
⑦ Remember This
MAC = local identity. Easy to fake, but only matters on the same network segment. Most switching attacks target MACs.

Router
The decision-maker that connects networks.
N·07routerlayer 3
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"A router decides which network a packet should go to next."
A router connects multiple networks together and decides where to send each packet. Operating at Layer 3, it reads destination IP addresses and forwards traffic toward its destination using a routing table.
② Explanation

Your home router actually combines several functions: routing, firewall, DHCP server, DNS resolver, and WiFi access point. Enterprise networks often separate these into different devices.

The internet is essentially a huge network of routers passing packets between each other. Each router knows about its neighbors and uses protocols like BGP to learn paths. If routing fails or is manipulated, traffic can be misdirected — a major attack vector against large infrastructure.

③ Keywords
routergatewayrouting tableNATBGPOSPFdefault gateway
④ Where It Appears
Home and office WiFi routers
ISP backbone equipment
Cloud VPCs and virtual networks
The core of the internet
⑤ How It Works
Each network interface on a router has its own IP. When a packet arrives, the router checks the destination IP against its routing table to find the next hop. It rewrites the source/destination MAC addresses (since the next hop is a different L2 segment), decrements TTL, and forwards. Packets without a matching route go to the default gateway — usually the upstream ISP.
⑥ Attack & Defense
Attack
  • Default credentials on home routers — admin/admin still works on millions
  • Outdated firmware with known CVEs
  • BGP hijacking — announce false routes to capture traffic at internet scale
  • DNS poisoning via compromised routers
Defense
  • Update firmware regularly (most home users never do)
  • Change default credentials immediately
  • Disable WAN-side admin access
  • For enterprises — RPKI to validate BGP routes
⑦ Remember This
Router compromise = full network compromise. Whoever controls routing controls who sees what. Default credentials are still the #1 router weakness.

Switch
The traffic director inside a network.
N·08switchlayer 2
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"A switch forwards data within a single network — by MAC, not IP."
A switch connects devices within the same network segment, forwarding frames based on MAC addresses. Unlike a hub (which broadcasts everything), a switch learns which devices are on which port and sends data only where it needs to go.
② Explanation

Switches are the unsung heroes of office and data center networks. They make wired networks work by intelligently routing frames — only the intended recipient sees the data, not every connected device.

Modern switches also support VLANs (logical separation of network segments), Quality of Service, and access control. Compromising a switch or tricking it gives an attacker visibility into traffic that would normally be invisible from elsewhere on the network.

③ Keywords
switchCAM tableVLANEthernetbroadcast domainSTP
④ Where It Appears
Office networks — physical and wireless
Data center spine and leaf
Home routers (which are also switches)
Industrial control networks
⑤ How It Works
Each port on the switch corresponds to a connected device. As traffic flows, the switch builds a CAM (Content Addressable Memory) table mapping MAC addresses to ports. When a frame arrives, it looks up the destination MAC and forwards only to that port. Unknown MACs cause the switch to flood the frame to all ports.
⑥ Attack & Defense
Attack
  • MAC flooding — fill the CAM table so it overflows and floods all traffic (sniffable)
  • VLAN hopping — exploit weak VLAN configuration to access other VLANs
  • Spanning Tree (STP) attacks — claim to be the root bridge and redirect traffic
Defense
  • Port security — limit number of MACs per port, sticky MAC
  • Disable unused ports physically and logically
  • Configure VLANs properly — never put untagged management on the same VLAN as users
⑦ Remember This
A switch makes networks fast AND private. Attacks usually try to make it forget — turning it back into a hub that broadcasts everything.

A firewall is a security device or software that filters network traffic based on rules. Some traffic is allowed, some blocked, some logged. It is the first line of defense between trusted and untrusted networks.
② Explanation

  • Bypass via allowed ports — tunnel forbidden traffic over HTTPS (443) which is almost always open

Ports & Protocols
The address system within an IP address.
N·10portsprotocols
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Port = a specific service on a host. IP gets you to the building, port gets you to the right office."
A port is a numbered communication endpoint on a device, ranging 0-65535. Combined with an IP address, it identifies a specific service. Common ports include SSH (22), HTTP (80), HTTPS (443), DNS (53). Port + protocol (TCP/UDP) defines what service runs there.
② Explanation

Ports below 1024 are "well-known" and require admin privileges to bind on most systems. These are reserved for standard services. Higher ports are "ephemeral" — temporary ports used by clients for outgoing connections.

For attackers, the first step is almost always port scanning — finding which ports are open on a target. Each open port is a potential attack surface. For defenders, the principle is the opposite: close every port that does not need to be open.

③ Keywords
portserviceTCPUDP22 SSH80 HTTP443 HTTPS53 DNS3389 RDP
④ Where It Appears
Network scanning with Nmap
Server configuration
Firewall rules
Application listening sockets
⑤ How It Works
When a service starts, it "binds" to a port and starts listening. When traffic arrives at that IP and port, the OS routes it to the application. Multiple services can run on different ports of the same host. Closed ports respond with RST (TCP) or ICMP unreachable. Filtered ports do not respond at all.
⑥ Attack & Defense
Attack
  • Port scanning to find open services (nmap, masscan)
  • Banner grabbing — connect to ports to fingerprint software versions
  • Exploit known vulnerabilities in services running on open ports
Defense
  • Close every unnecessary port — minimize attack surface
  • Use non-standard ports for SSH and admin services (security through obscurity is weak alone, but useful as an additional layer)
  • Monitor for port scans — alert on unusual scan patterns
⑦ Remember This
Open port = entrance. Every open port should justify its existence. The first thing every attacker does is enumerate ports.

Packet
The atom of network communication.
N·11packetfundamentals
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Every byte you send becomes a packet — a small bundle with destination info."
A packet is a small unit of data that travels independently across a network. It contains a payload (the actual data) and headers (metadata like source IP, destination IP, port, protocol). Big files become many packets that get reassembled at the destination.
② Explanation

When you send a 100MB file, the network does not send 100MB at once. It is broken into many packets — each typically 1500 bytes (the standard MTU). Each packet finds its own way through the network and they may even arrive out of order. The destination puts them back together.

This packet-based design is what makes the internet resilient. If one path fails, packets find another. It is also what makes networks "sniffable" — packets travel in cleartext (unless encrypted) and anyone with access to the path can read them.

③ Keywords
packetheaderpayloadfragmentMTUframedatagram
④ Where It Appears
Wireshark and packet captures
Network troubleshooting
Forensic analysis
Detection of malicious traffic patterns
⑤ How It Works
A packet is built layer by layer (encapsulation): the application data is wrapped with TCP/UDP header, then IP header, then Ethernet header. Each layer adds its own metadata. At the destination, headers are peeled off in reverse — Ethernet stripped, then IP, then TCP/UDP, leaving the original data.
⑥ Attack & Defense
Attack
  • Sniffing — capture packets traversing the network (especially on shared media)
  • Packet injection — inject malicious packets to manipulate connections
  • Replay attacks — capture legitimate packets and resend them later
Defense
  • Encrypt all traffic — TLS, SSH, VPN make sniffed packets useless
  • Switched networks — much harder to sniff than older hub-based networks
  • IDS/IPS — detect anomalous packet patterns
⑦ Remember This
Packets are fundamental — every attack and defense ultimately operates on packets. Wireshark shows you exactly what is on the wire.

Bandwidth
How much data can flow per second.
N·12bandwidthcapacity
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Bandwidth is the size of the pipe. More bandwidth = more simultaneous data."
Bandwidth is the maximum amount of data that can transfer through a network connection in a given time, usually measured in bits per second (Mbps, Gbps). It is the network capacity — not how fast individual packets travel, but how many can flow at once.
② Explanation

Confusing terms: bandwidth is capacity, throughput is actual achieved transfer rate, latency is delay between request and response. A connection can have high bandwidth but high latency (satellite internet) or low bandwidth but low latency (old DSL).

In security, bandwidth matters because exhausting it is the entire goal of volumetric DDoS attacks. If a target has 1 Gbps of bandwidth and an attacker can flood 10 Gbps of garbage, legitimate traffic cannot get through. The defender does not need to block the attack — they just need to absorb it.

③ Keywords
bandwidththroughputMbpsGbpscapacitysaturationDDoS
④ Where It Appears
Internet speed measurements
Cloud cost calculations (egress is billed)
Network design and capacity planning
DDoS attack metrics
⑤ How It Works
Networks have physical maximums dictated by the medium (copper, fiber, wireless) and the equipment. Beyond that, congestion management kicks in — packets get queued or dropped. Bandwidth is shared among users on a link, so 100 Mbps shared by 100 users gives 1 Mbps each in worst case.
⑥ Attack & Defense
Attack
  • Volumetric DDoS — flood the target with more traffic than its bandwidth can absorb
  • Amplification attacks — use protocols like DNS or NTP to multiply attack volume
  • Slowloris — opposite of volumetric, exhaust connections without high bandwidth
Defense
  • DDoS protection services (Cloudflare, AWS Shield) absorb attacks at scale
  • Anycast networks distribute traffic across many points of presence
  • Rate limiting and traffic shaping at the perimeter
⑦ Remember This
Bandwidth is the bottleneck. Most "the website is down" incidents are bandwidth saturation, whether from attack or unexpected legitimate traffic.

Proxy

A proxy is a server that sits between a client and another server, forwarding requests and responses. It can hide the client's identity, cache content, filter content, or inspect traffic. Forward proxies act on behalf of clients; reverse proxies act on behalf of servers.

Encrypted tunnel between two networks.

"A VPN creates a secure tunnel — your traffic looks like it came from somewhere else."
A VPN (Virtual Private Network) creates an encrypted tunnel between your device and a VPN server. All your traffic flows through this tunnel. NAT
Network Address Translation — making many appear as one.
N·15NATaddress translation
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"NAT lets multiple devices share one public IP — and accidentally provides some protection."
NAT (Network Address Translation) is the technique of mapping multiple private IP addresses to a single public IP address. Almost every home router does this — your phone, laptop, and TV all share one public IP from your ISP. NAT was invented because IPv4 ran out of addresses.
② Explanation

NAT works by tracking outbound connections. When your laptop (192.168.1.5) sends to google.com, the router rewrites the source IP to its public IP and remembers the mapping. When Google replies, the router uses the mapping to send the response back to your laptop.

A side effect of NAT: incoming connections from the internet have nowhere to go by default — they would not know which internal device to send to. This provides incidental security (often confused with a real firewall). NAT is not a security control, but it does break direct inbound attacks.

③ Keywords
NATPATprivate IPpublic IPport forwardingIPv4 exhaustionCGN
④ Where It Appears
Home routers (always)
Corporate networks behind firewalls
Cloud VPCs with public IPs assigned to gateways
Cellular networks (carrier-grade NAT)
⑤ How It Works
Router maintains a translation table mapping (internal IP, internal port) ↔ (external IP, external port). Outbound: rewrite source IP/port, log mapping. Inbound: look up destination IP/port in mapping table, rewrite to internal IP/port. Without an existing mapping, incoming packets are dropped (unless explicitly forwarded).
⑥ Attack & Defense
Attack
  • NAT does not block outbound — malware on internal devices can connect outbound freely
  • Port forwarding misconfigurations expose internal services
  • UPnP — devices automatically punch holes in NAT (often dangerous)
Defense
  • Disable UPnP on routers — most users do not need it
  • Use a real firewall — NAT is not a security boundary
  • Egress filtering — control what can leave the network
⑦ Remember This
NAT = translation, not protection. The "security" of NAT is incidental. Always use a real firewall — never rely on NAT to keep attackers out.

Subnet
Dividing networks into smaller, manageable pieces.
N·16subnetCIDR
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"A subnet is a chunk of IP space carved out for a specific group."
A subnet (subnetwork) is a logical division of a larger network. Within a subnet, devices can communicate directly without going through a router. Subnets are defined by CIDR notation like 192.168.1.0/24, where /24 means the first 24 bits identify the network and remaining 8 bits identify hosts (256 addresses).
② Explanation

Subnetting is critical for security. Network segmentation divides infrastructure into zones (web servers, database servers, dev/staging/prod) so a breach in one zone does not automatically reach others. Without subnetting, a flat network means lateral movement is trivial once an attacker gets in.

CIDR math: /24 = 256 addresses, /16 = 65536 addresses, /8 = 16M addresses. Larger numbers mean smaller subnets. Common defaults: home networks /24, corporate departments /22, large enterprises break into many subnets connected by routers.

③ Keywords
subnetCIDRsubnet mask/24segmentationbroadcast domainVLAN
④ Where It Appears
Network design diagrams
Cloud VPC configuration
Firewall rules (allow from subnet X to subnet Y)
Penetration testing (pivoting between subnets)
⑤ How It Works
Each device on a subnet shares a network prefix (the bits to the left of the / in CIDR). Devices use the subnet mask to determine if a destination IP is local (same subnet — send directly) or remote (different subnet — send to gateway). Broadcast traffic stays within the subnet.
⑥ Attack & Defense
Attack
  • Flat networks = total compromise from any breach point — attacker can reach everything
  • Misconfigured subnet boundaries — overly permissive rules between zones
  • ARP spoofing within a subnet to intercept traffic
Defense
  • Microsegmentation — many small subnets with strict rules between them
  • Network Access Control (NAC) for endpoint posture
  • Zero trust architecture — verify every cross-subnet request
⑦ Remember This
Subnetting = damage control. When (not if) something gets compromised, good subnetting limits how far the damage spreads.

DHCP
Automatic IP address assignment.
N·17DHCPauto-config
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"DHCP is how devices get an IP address without you typing one in."
DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses, subnet masks, default gateways, and DNS servers to devices joining a network. When your phone connects to WiFi, DHCP gives it everything it needs — instantly.
② Explanation

DHCP works through a four-step "DORA" process: Discover (client broadcasts looking for DHCP), Offer (server offers an IP), Request (client formally requests it), Acknowledge (server confirms and assigns). The client gets a lease — a temporary right to use the IP for some time.

Without DHCP, every device would need manual configuration. With DHCP, devices come and go from networks seamlessly. But DHCP is also a juicy attack vector — controlling DHCP means controlling network configuration for everyone connecting.

③ Keywords
DHCPDORAleasebroadcastgatewaystatic IPreservation
④ Where It Appears
Home and office WiFi networks
Corporate wired networks
Conference and event WiFi
Cellular networks (a similar process)
⑤ How It Works
When a device joins, it broadcasts a DISCOVER. Any DHCP server on the network may respond with an OFFER. The client picks one and sends a REQUEST for the offered IP. The server confirms with ACK and records the lease (time-limited). The client starts using the IP. Before the lease expires, the client renews. If the device leaves, the IP eventually becomes available again.
⑥ Attack & Defense
Attack
  • Rogue DHCP server — set up unauthorized DHCP that hands out malicious DNS or gateway, redirecting traffic
  • DHCP starvation — exhaust the pool by requesting all available IPs, denying service to legitimate users
  • Snooping the DHCP traffic to learn about the network
Defense
  • DHCP snooping on switches — only allow DHCP responses from authorized servers
  • DHCP authentication and isolation
  • Monitor for unauthorized DHCP servers on the network
⑦ Remember This
DHCP is invisible until something goes wrong. Rogue DHCP servers are sneaky and effective — they redirect traffic before any application-layer security kicks in.

Kernel
The core that talks to hardware.
L·03kernelcore
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"The kernel is the lowest layer of the OS — everything else sits on top."
The kernel is the central program that manages hardware resources and provides services to user programs. When you run a program, it actually asks the kernel to do everything that touches hardware — read files, send packets, allocate memory.
② Explanation

User programs cannot talk to hardware directly — they make "system calls" (syscalls) to the kernel. The kernel decides what is allowed, schedules processes, manages memory. This separation is what keeps the system stable: a crashing app does not crash the kernel.

Linux kernel is monolithic — drivers run inside it for speed. This means a buggy driver can crash everything, but performance is excellent. Loadable modules let you add functionality without rebooting.

③ Keywords

/proc/version shows your kernel
uname -a output
Kernel CVE databases
Driver loading and kernel module errors
⑤ How It Works
When a process needs anything from the system (open file, send packet, allocate RAM), it triggers a syscall. CPU switches from userspace to kernelspace, kernel handles the request, returns result, switches back. Hundreds of syscalls happen per second per process. Kernel uses scheduling algorithms to share CPU among processes.
⑥ Attack & Defense
Attack
  • Kernel exploits — vulnerabilities in syscalls or drivers give attacker root
  • Loadable kernel rootkits hide as kernel modules
  • Outdated kernels have known privilege escalation bugs (Dirty Pipe, Dirty COW)
Defense
  • Keep kernel updated — security patches are critical
  • Restrict kernel module loading on production systems
  • Use kernel hardening (SELinux, AppArmor)
⑦ Remember This

"Linux file system is one big tree starting at / (root)."
In Linux, all files live under a single root directory called /. There are no drive letters like C: or D:. Different storage devices "mount" into directories — your USB stick might appear at /mnt/usb. This unified tree is one of Linux's defining features.
② Explanation

Standard directories: /etc (configs), /var (variable data like logs), /home (user files), /usr (user programs), /bin (essential binaries), /tmp (temporary files), /proc (kernel info as virtual files), /dev (devices). Knowing where things live lets you navigate any Linux system fast.

In security, sensitive paths to remember: /etc/passwd (user list), /etc/shadow (password hashes — root only), /var/log (system logs), /root (root's home), /home/* (user homes). These are the first places attackers and defenders look.

③ Keywords
filesystemroot //etc/var/log/etc/shadow/tmpmountFHS
④ Where It Appears
Every directory you cd into
Linux Filesystem Hierarchy Standard (FHS)
Forensics during investigations
Privilege escalation enumeration
⑤ How It Works
When you boot, the kernel mounts the root filesystem (/) from disk. Other partitions, USB drives, network shares get mounted at specific paths. ls / shows top-level directories. Each directory may live on a different physical device but they all appear as one tree. Mount points are tracked in /etc/fstab and /proc/mounts.
⑥ Attack & Defense
Attack
  • World-writable directories like /tmp can be abused to plant malicious files
  • Misconfigured mount points (NFS shares with bad permissions) leak data
  • Accessing /etc/shadow with read access = offline password cracking party
Defense
  • Strict permissions on sensitive paths — /etc/shadow should be 600 root:root
  • Mount /tmp with noexec to prevent execution from there
  • Audit /etc and /var regularly for unauthorized changes
⑦ Remember This
In Linux everything is a file, and every file has a path starting from /. Master the tree and you master the system.

Permissions
rwx for owner, group, and others.
L·07permissionsrwx
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Linux permissions: who can read, write, or execute each file."
Permissions in Linux control file access via three classes (owner, group, others) and three actions (read, write, execute). Shown as -rwxr-xr-- in ls -l. Numeric form: 754 (each digit is a 3-bit value).
② Explanation

Reading rwxr-xr--: owner can read+write+execute, group can read+execute, others can only read. Numeric: r=4, w=2, x=1. So 7=rwx, 5=r-x, 4=r--. Memorize this and you can read any chmod number at a glance.

Special bits matter for security: SUID (4xxx) makes a file run as its owner regardless of who runs it. SGID (2xxx) similar but for group. Sticky bit (1xxx) on /tmp prevents users from deleting each other's files. SUID files owned by root are the #1 path to privilege escalation in CTFs.

③ Keywords
permissionsrwxchmodchownSUIDSGIDsticky755
④ Where It Appears
ls -l output everywhere
chmod commands
Privilege escalation enumeration (find / -perm -4000)
Hardening guides
⑤ How It Works
Each file has owner UID, group GID, and a 12-bit permission word. When you try to access a file, kernel checks: are you the owner? Use owner perms. Are you in the group? Use group perms. Otherwise use other perms. SUID bit elevates the executing process to file owner's privileges during execution.
⑥ Attack & Defense
Attack
  • SUID-root binaries with vulnerabilities = root shell (check GTFOBins)
  • World-writable files in PATH let attacker replace them
  • Misconfigured shared directories let users tamper with each other's data
Defense
  • Principle of least privilege — never chmod 777 anything
  • Audit SUID files regularly: find / -perm -4000 -ls
  • Use ACLs for fine-grained control beyond rwx
⑦ Remember This
rwx = 4+2+1. Memorize the numbers. SUID owned by root is what every Linux pentest checks first.

User & Group
Identity in Linux: UID and GID.
L·08usersgroups
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Every process and file has a user (UID) and group (GID) attached."
In Linux, every action is done by a user, identified by a numeric UID. User 0 is special — that's root, with unlimited power. Users belong to one or more groups, identified by GIDs. Permissions check both.
② Explanation

User info lives in /etc/passwd (publicly readable, hashes are NOT here despite the name). Password hashes live in /etc/shadow (root-only). Group definitions in /etc/group. The "wheel" or "sudo" group typically grants sudo rights.

Service accounts (www-data, nginx, mysql) have low UIDs and limited shells (/usr/sbin/nologin). They run services without giving them more access than needed. A web server compromised should only have www-data privileges, not root — that is by design.

③ Keywords
usergroupUIDGIDroot/etc/passwd/etc/shadowsudo
④ Where It Appears
whoami and id commands
/etc/passwd file inspections
sudoers configuration
Privilege escalation enumeration
⑤ How It Works
When you log in, the system creates a session under your UID. Every program you start inherits that UID and your group memberships. File access checks compare your UID/GID with the file's owner/group. Setuid programs temporarily change to file owner UID during execution.
⑥ Attack & Defense
Attack
  • Cracking /etc/shadow hashes offline if you can read it
  • Adding yourself to powerful groups (docker = root via container escape)
  • Manipulating sudoers for free root commands
Defense
  • Strong password policies — bcrypt or argon2 in /etc/shadow
  • Restrict sudo carefully — never NOPASSWD on dangerous commands
  • Disable unused accounts and remove orphan UIDs
⑦ Remember This
UID 0 = root = god mode. Everything in privilege escalation is about going from a low UID to UID 0.

sudo
Run a single command with root privileges.
L·09sudoprivesc
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"sudo lets a regular user run specific commands as root, with their own password."
sudo ("substitute user do") allows authorized users to run commands as another user (typically root). Configured via /etc/sudoers. Logs every use. Designed so admins do not have to share root password or stay logged in as root.
② Explanation

Run sudo -l to see what you can run as root without typing the root password. This is one of the first commands any pentester runs after getting shell access. The output reveals your potential paths to escalation.

sudoers entries can be specific (user can run /usr/bin/apt only) or broad ((ALL) ALL = run anything as root). NOPASSWD removes the password requirement. Misconfigured sudoers is one of the most common privilege escalation vectors — even vim or find with sudo can give you root if allowed.

③ Keywords
sudosudoersNOPASSWD/etc/sudoersvisudosudo -lGTFOBins
④ Where It Appears
Daily admin work
Sudo logs in /var/log/auth.log
Pentest enumeration (sudo -l)
CTF privilege escalation
⑤ How It Works
When you run sudo command, sudo checks /etc/sudoers for a matching rule. If allowed, prompts for your password (cached for ~15 min). If approved, fork+exec the command as the target user. Logs everything to syslog. Specific commands can be allowed without password using NOPASSWD directive.
⑥ Attack & Defense
Attack
  • (ALL) NOPASSWD: ALL = full root, no password — instant takeover if you get shell
  • Allowed commands like vim, find, awk, less can break out to a shell with sudo (GTFOBins)
  • PATH-based sudo rules (relative paths) can be hijacked
Defense
  • Always require password unless absolutely necessary
  • Limit sudo rules to specific commands needed
  • Use full absolute paths in sudoers entries
⑦ Remember This
sudo -l is your map to root. Always check it. GTFOBins shows you how to abuse any sudo-allowed command.

Process Management
Watching and controlling running programs.
L·10processesps
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Linux gives you full visibility and control over every running process."
Process management covers starting, stopping, monitoring, and signaling running programs. Tools: ps (list), top/htop (live view), kill (signal), nohup (detach), & (background).
② Explanation

Every process has a PID, parent PID (PPID), owner (UID), state, memory usage, and CPU time. ps aux shows all processes with details. The ps output is structured: USER PID %CPU %MEM ... COMMAND.

Signals are how processes get notified. SIGTERM (15) asks politely to stop. SIGKILL (9) cannot be ignored — kernel kills the process. SIGHUP (1) is often used to make daemons reload configs. Knowing process state and signals is essential for both administrators and attackers.

③ Keywords
processpstophtopkillPIDPPIDsignalsystemd
④ Where It Appears
ps aux for full process list
top for live monitoring
systemctl to manage services
Reverse shell post-exploitation
⑤ How It Works
When you run a program, the kernel creates a process with a unique PID. Each process has its own memory space, file descriptors, and credentials. ps reads /proc/[PID]/ entries which the kernel maintains. systemd manages long-running services. nohup detaches a command so it survives terminal close.
⑥ Attack & Defense
Attack
  • Process injection — load malicious code into legitimate process memory
  • Hide malware as innocuous-named processes (kworker, systemd-helper)
  • Persistence via cron jobs or systemd services
Defense
  • Audit running processes regularly — anything unfamiliar deserves investigation
  • Process monitoring tools (auditd, sysmon for Linux)
  • Lock down systemd unit file permissions
⑦ Remember This
ps aux first, ask questions later. A processes you cannot account for is the loudest red flag on a compromised system.

Package Manager
How you install software in Linux.
L·11aptdnfpacman
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Package managers install, update, and remove software with one command."
A package manager handles software installation in Linux. Different distros use different ones: apt (Debian/Ubuntu/Kali), dnf/yum (Fedora/RHEL/CentOS), pacman (Arch). They handle dependencies, updates, and uninstalls.
② Explanation

Why this matters in security: package managers verify cryptographic signatures so you know packages came from trusted sources. They track every installed file. apt update + apt upgrade is how you get security patches.

For pentesters, package managers are also reconnaissance tools. dpkg -l (or rpm -qa) shows everything installed and exact versions, instantly revealing exploitable software. Many CVEs are version-specific — older packages mean known exploits.

③ Keywords
aptdnfyumpacmanrpmdpkgrepositorypackagedependency
④ Where It Appears
Daily software installation
apt install / yum install commands
Security updates
CVE checks during pentests
⑤ How It Works
When you run apt install nginx, apt: queries package lists from /etc/apt/sources.list, finds nginx and its dependencies, downloads .deb packages, verifies GPG signatures, runs pre-install scripts, copies files to standard locations, runs post-install scripts, updates package database. dpkg is the lower-level tool apt uses.
⑥ Attack & Defense
Attack
  • Outdated packages with known CVEs — first thing to enumerate (dpkg -l, rpm -qa)
  • Compromised repositories distribute malicious updates (rare but happens)
  • Adding malicious repository to install backdoored packages
Defense
  • apt update && apt upgrade regularly — automate when possible
  • Verify GPG signatures on packages — never disable
  • Pin trusted repository versions and minimize third-party repos
⑦ Remember This
Update or get owned. Most servers fall to known CVEs in unpatched packages — not zero-days. apt upgrade is your best friend.

Logs
Where Linux records everything that happens.
L·12logs/var/log
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Logs are the timeline of your system — read them when things go wrong."
Logs are text files where the system, services, and apps record events. Most live in /var/log/. Critical files: auth.log (logins, sudo), syslog (general), kern.log (kernel), Apache/nginx logs. Modern systems use journalctl too.
② Explanation

When something breaks or you suspect compromise, logs are where you start. Failed SSH attempts, sudo abuse, kernel errors, application crashes — all recorded. Reading logs is a core skill in both administration and forensics.

For attackers, logs are evidence to remove. Clearing /var/log/auth.log after compromise is classic. For defenders, this is why centralized logging (syslog server, SIEM) matters — copies of logs leave the compromised host immediately. Once the attacker is in, you can't trust local logs.

③ Keywords
logs/var/logauth.logsyslogjournalctlkern.logrsysloglogrotate
④ Where It Appears
Troubleshooting any failure
Forensic investigations
IDS/SIEM ingestion
Compliance audits
⑤ How It Works
Programs and the kernel write events to log files via syslog or systemd-journal. Logs rotate periodically — old entries are compressed and archived (logrotate). On modern systems, journalctl shows binary journals from systemd. Filtering: tail -f for live, grep for searching, awk for parsing.
⑥ Attack & Defense
Attack
  • Log clearing after compromise — wipe auth.log to hide your tracks
  • Log injection — control input that gets logged unsanitized
  • Logrotate exploits and log file race conditions
Defense
  • Centralize logs — send to a separate, hardened log server
  • Make logs append-only when possible (chattr +a)
  • Monitor for log volume drops (sudden silence is suspicious)
⑦ Remember This
Logs answer "what happened?" Read them often. Centralize them religiously. The attacker who wipes /var/log/auth.log on the compromised host has nothing on the SIEM that already received the entries.

SSH
Secure remote shell — the lifeline of every server.
L·13sshremote
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"SSH gives you encrypted shell access to a remote machine."
SSH (Secure Shell) is the standard protocol for secure remote access. Connects encrypted to a remote machine and gives you shell access. Listens on TCP port 22 by default. Replaces insecure protocols like telnet (everything in cleartext).
② Explanation

SSH supports password auth, key-based auth (vastly safer), and certificate auth. Public/private key pairs are the standard: keep the private key secret, put the public key on every server you want to access. Add a passphrase to the private key for extra protection.

SSH is also a tunnel. ssh -L lets you forward ports — useful for accessing services behind firewalls. Reverse SSH tunnels (ssh -R) are how attackers maintain persistence after a breach. SSH is one of the most powerful tools in any sysadmin's and attacker's arsenal.

③ Keywords
SSHport 22private keypublic key~/.sshssh-keygentunnelauthorized_keys
④ Where It Appears
Remote server administration
Git over SSH
Persistence via SSH keys
Tunneling and pivoting
⑤ How It Works
Client connects to server on port 22. Both negotiate encryption algorithms. Server presents its public key (verified against ~/.ssh/known_hosts). Client authenticates (password or key). Once authenticated, all traffic flows through the encrypted tunnel — you have a shell as remote user. Keys live in ~/.ssh/ on both ends.
⑥ Attack & Defense
Attack
  • Bruteforce SSH passwords on port 22 (always exposed somewhere)
  • Steal SSH private keys from compromised laptop = access to every server they touch
  • Persistence via authorized_keys — drop attacker's public key for permanent access
Defense
  • Disable password auth — keys only
  • Use strong key passphrases
  • Move SSH off port 22 (security through obscurity, but reduces noise)
  • Monitor authorized_keys for unauthorized entries
⑦ Remember This
SSH keys = your kingdom. Lose them and you lose every server. Audit ~/.ssh/authorized_keys regularly on every host.

Cron
Scheduling tasks to run automatically.
L·14cronscheduling
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"Cron runs commands on schedule — backups, cleanups, anything repetitive."
cron is the standard Linux task scheduler. Each user has a crontab (cron table) listing scheduled commands. The cron daemon runs them at the specified times. Format: 5 time fields + command. 0 3 * * * means "3 AM every day".
② Explanation

Each user can have a crontab via crontab -e. The system also has /etc/crontab and /etc/cron.d/, plus shorthand directories /etc/cron.daily, /etc/cron.weekly, /etc/cron.monthly that run scripts on those schedules.

cron is everywhere because automation needs scheduling. It is also a favorite for attackers seeking persistence. A cron job that runs every 5 minutes as root, downloading and executing a payload, is a classic backdoor. Reviewing cron entries is essential during incident response.

③ Keywords
croncrontabscheduling/etc/crontabcron.dailypersistencesystemd timer
④ Where It Appears
Backups, log rotation, system maintenance
CTF privilege escalation paths
Persistence after compromise
Modern systems also use systemd timers
⑤ How It Works
cron daemon (crond) runs in the background. Every minute it checks all crontabs (user and system). If a job's schedule matches the current time, cron forks and runs the command as the appropriate user. Output is mailed to the user (or discarded if mail is not configured).
⑥ Attack & Defense
Attack
  • Persistence — drop attacker payload in /etc/cron.d or user crontab
  • Cron jobs running as root with writable scripts = instant root
  • Race conditions in cron scripts can be exploited
Defense
  • Audit cron jobs regularly — list all of them across users
  • Restrict cron access (cron.allow / cron.deny)
  • Make cron job scripts read-only and owned by root
⑦ Remember This
cron jobs are an attacker's favorite persistence mechanism. After any incident, walk every crontab — especially /etc/cron.* and root's.

systemd
The init system and service manager.
L·15systemdservices
tags: Estudado Revisar Difícil Favorito
① Simple Definition
"systemd starts everything when you boot and manages services while running."
systemd is the modern init system used by most Linux distros. It is the first process started by the kernel (PID 1) and starts everything else. It also manages services, timers, mounts, and more. Replaces older SysV init.
② Explanation

Services are defined in unit files (/etc/systemd/system/*.service or /lib/systemd/system/*.service). systemctl is the tool: systemctl start nginx, systemctl status, systemctl enable. Logs go to journalctl.

systemd is controversial (mostly because it does many things) but it is the reality on Ubuntu, Debian, Fedora, RHEL, Arch — practically everywhere. Knowing systemctl and journalctl is essential. systemd timers are also replacing cron in many cases — more flexible and integrated with logging.

③ Keywords
systemdsystemctlunit fileservicejournalctl.serviceenablemask
④ Where It Appears
Service management on modern Linux
systemctl status checks during admin work
journalctl for system logs
Persistence via malicious service units
⑤ How It Works
systemd reads unit files defining services. systemctl talks to the systemd daemon to start, stop, enable services. Enabled services start at boot. Each service runs as its own process supervised by systemd — if it crashes, systemd can restart it. journalctl reads systemd's binary journal for logs.
⑥ Attack & Defense
Attack
  • Persistence — drop a malicious .service file with WantedBy=multi-user.target for boot persistence
  • Services running as root with writable ExecStart paths can be hijacked
  • Disabling security-related services (auditd, falco) to operate undetected
Defense
  • Audit /etc/systemd/system/ regularly for unfamiliar units
  • Lock down unit file permissions — root only writable
  • Use systemd security directives (PrivateTmp, NoNewPrivileges) for services
⑦ Remember This
systemctl + journalctl is the modern admin toolkit. Master both. Malicious systemd units are the new cron jobs for persistence.
