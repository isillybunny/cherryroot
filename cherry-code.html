Programming
Escrever instruções que um computador consegue executar.
F·01programmingbasics
tags: Estudado Revisar Difícil Favorito
① Definição
"Programar é dizer ao computador o que fazer, passo a passo."
Código é uma sequência precisa de instruções numa linguagem que o computador consegue interpretar. Todo app, site e jogo é, no fundo, uma série de instruções de programação.
② Palavras-chave
codeinstructionslogicexecutionalgorithm
③ Como Funciona
Você escreve o código → o computador lê → executa os passos um a um. Programas resolvem problemas quebrando-os em instruções pequenas e ordenadas.
④ Exemplo de Código
# Your first program
print("Hello, World!")
// Your first program
console.log("Hello, World!");
public class Main {
  public static void main(String[] args) {
    System.out.println("Hello, World!");
  }
}
⑤ Lembre-se
Código é comunicação — entre você e o computador, E entre você e o você do futuro (ou sua equipe).

Code
O texto que escrevemos numa linguagem de programação.
F·02codesource
tags: Estudado Revisar Difícil Favorito
① Definição
"Código é o texto de um programa — escrito numa linguagem específica com regras estritas."
O código-fonte é a versão legível por humanos de um programa. Ferramentas (interpretadores/compiladores) convertem isso em instruções que a máquina realmente roda. Código vive em arquivos com extensões como .py, .js, .java.
② Palavras-chave
source.py.js.javasyntaxlanguage
③ Como Funciona
Você escreve código num editor de texto → salva como um arquivo → roda com a ferramenta certa. Código pode ser uma única linha ou milhões de linhas espalhadas por milhares de arquivos.
④ Exemplo de Código
# File: greet.py
name = "Cherry"
print(f"Hello, {name}!")
// File: greet.js
const name = "Cherry";
console.log(`Hello, ${name}!`);
// File: Greet.java
String name = "Cherry";
System.out.println("Hello, " + name);
⑤ Lembre-se
Código é texto. Só isso. Arquivos de texto que seguem regras estritas.

cherry code › F·03
Compiler vs Interpreter
Duas formas de transformar código em algo que o computador roda.
F·03compilerinterpreter
tags: Estudado Revisar Difícil Favorito
① Definição
"Compiladores traduzem tudo antes de rodar. Interpretadores traduzem linha por linha conforme avançam."
Um compiler converte o programa inteiro em código de máquina de uma vez, depois roda o resultado (rápido, mas demora pra começar). Um interpreter lê e executa linha por linha (mais lento em runtime, mas com feedback instantâneo). Java é compilado. Python e JavaScript são interpretados.
② Palavras-chave
compileinterpretbytecoderuntimeJIT
③ Como Funciona
Compiler: fonte → arquivo executável → roda.
Interpreter: fonte → traduzido E executado linha por linha.
Linguagens modernas misturam os dois (compilação JIT).
④ Exemplo de Código
# Interpreted — run directly
# $ python script.py
print("runs line by line")
// Interpreted by the browser/Node
// No compile step needed
console.log("runs immediately");
// Compiled FIRST: javac Main.java
// THEN run: java Main
System.out.println("compiled then run");
⑤ Lembre-se
Python/JS = digitar e rodar. Java = compilar, depois rodar. No fim, os dois acabam fazendo a mesma coisa.

Bug
Quando o código não faz o que você queria.
F·04bugerror
tags: Estudado Revisar Difícil Favorito
① Definição
"Um bug é qualquer erro no código que causa comportamento errado ou travamentos."
Bugs são inevitáveis — todo programador cria bugs. A habilidade não é evitá-los, mas ACHAR e CORRIGIR rápido. Tipos: erros de sintaxe (nem rodam), erros de runtime (travam no meio da execução), bugs de lógica (rodam, mas com saída errada).
② Palavras-chave
bugerrorcrashlogicedge case
③ Como Funciona
Você escreveu algo do jeito que pretendia — mas o computador lê exatamente o que você digitou. Bug = a distância entre o que você digitou e o que você quis dizer.
④ Exemplo de Código
# BUG: wrong operator
age = 18
if age = 18:  # SyntaxError: should be ==
    print("adult")
// BUG: typo causes ReferenceError
let name = "Cherry";
console.log(nme);  // nme is not defined
// BUG: array out of bounds
int[] nums = {1, 2, 3};
System.out.println(nums[5]);  // crashes!
⑤ Lembre-se
Todo dev sênior ainda cria bugs diariamente. Eles só ficaram mais rápidos em achá-los.

cherry code › F·05
Debug
Achar e corrigir bugs.
F·05debugfix
tags: Estudado Revisar Difícil Favorito
① Definição
"Debugar é o processo de investigar POR QUE o código se comporta mal e corrigir isso."
Passos: 1) Reproduzir o bug de forma consistente. 2) Ler a mensagem de erro (ela costuma dizer exatamente o que está errado). 3) Imprimir valores intermediários pra achar onde a lógica diverge. 4) Corrigir e testar. A maior parte do tempo programando é debugar, não escrever.
② Palavras-chave
debugbreakpointlogtracereproduce
③ Como Funciona
Adicione prints/logs ao redor do código suspeito → rode → veja quais valores realmente passam por ali → encontre a diferença em relação ao que você esperava → corrija.
④ Exemplo de Código
# Add print to debug
result = total / count
print(f"DEBUG: total={total}, count={count}")
# Oh! count is 0 → ZeroDivisionError
// console.log() is your friend
const result = total / count;
console.log("DEBUG:", { total, count });
// → { total: 100, count: 0 }
// Use println for quick debug
System.out.println("DEBUG total: " + total);
System.out.println("DEBUG count: " + count);
⑤ Lembre-se
Leia a mensagem de erro PRIMEIRO. É a frase mais importante na tela.

Terminal
Interface de texto pra rodar comandos.
F·07terminalCLI
tags: Estudado Revisar Difícil Favorito
① Definição
"O terminal é onde você digita comandos em vez de clicar em botões."
Essencial pra quem desenvolve: rodar scripts, instalar pacotes, usar git, fazer deploy de apps. Parece intimidante, mas você só precisa de uns 10 comandos no dia a dia. Windows: PowerShell/Terminal. Mac/Linux: Terminal.
② Palavras-chave
CLIshellbashPowerShellcommand
③ Como Funciona

④ Exemplo de Código
# Run a Python file
$ python script.py
# Install a package

# Interactive Python
$ python
# Run a JS file with Node
$ node app.js
# Install package
$ npm install express
# Start a project
$ npm start
# Compile a Java file
$ javac Main.java
# Run compiled class
$ java Main
# With Maven
$ mvn run
⑤ Lembre-se
O terminal é intimidante por 2 semanas, depois fica mais rápido que o mouse pra sempre.

Syntax
As regras de gramática de uma linguagem de programação.
F·08syntaxgrammar
tags: Estudado Revisar Difícil Favorito
① Definição
"Sintaxe é o jeito exato como você precisa escrever o código — sintaxe errada = o código não roda."
Como nos idiomas falados, a programação tem regras. Python usa indentação. JavaScript usa { }. Java exige ponto e vírgula depois de cada instrução. Erros de sintaxe são os bugs mais fáceis de achar — o compiler/interpreter literalmente diz onde eles estão.
② Palavras-chave

Function
A reusable named block of code that performs a task.
N·01functionsreuse
tags: Estudado Revisar Difícil Favorito
① Definição
"A function is a named, reusable block of code that performs a specific task."
You define it once and call it as many times as you need. Functions take inputs (parameters), do work, and optionally return a result.
② Palavras-chave
deffunctionreturncallinvokereuse
③ Como Funciona
Define a function with a name, parameters, and body. Call it by writing the name followed by parentheses with arguments. The function executes its body and may return a value back to the caller.
④ Exemplo de Código
def greet(name):
    return f"Hello, {name}!"

print(greet("Ada"))   # Hello, Ada!
function greet(name) {
  return `Hello, ${name}!`;
}
console.log(greet("Ada"));  // Hello, Ada!
static String greet(String name) {
    return "Hello, " + name + "!";
}
System.out.println(greet("Ada"));
⑤ Lembre-se
A good function does one thing. If you find yourself writing "and" in its name, split it in two.

"The return value is the result a function produces and sends back to the caller."

cherry code › N·05
Pure Function
Same inputs always produce the same outputs, with no side effects.
N·05functionsfunctional
tags: Estudado Revisar Difícil Favorito
① Definição
"A pure function depends only on its inputs and produces no observable side effects."
It does not modify external state, write files, print, or alter its arguments. Given the same inputs, it always returns the same output.
② Palavras-chave
puredeterministicside effectsfunctionaltestable
③ Como Funciona
Pure functions are easy to test, easy to cache, and safe to parallelize because they never reach outside themselves. Impure functions touch the world — they read clocks, mutate globals, or perform I/O.
④ Exemplo de Código
# Pure: depends only on inputs
def add(a, b):
    return a + b

# Impure: reads external state
counter = 0
def tick():
    global counter
    counter += 1
    return counter
// Pure
const add = (a, b) => a + b;

// Impure: mutates outside state
let counter = 0;
const tick = () => ++counter;
// Pure
static int add(int a, int b) {
    return a + b;
}

// Impure: depends on system time
static long now() {
    return System.currentTimeMillis();
}
⑤ Lembre-se
Prefer pure functions for the core logic; isolate impure code at the edges of your app.

cherry code › N·06
Recursion
A function that calls itself to solve a smaller version of the problem.
N·06functionsrecursion
tags: Estudado Revisar Difícil Favorito
① Definição
"Recursion is when a function calls itself, breaking the problem into smaller subproblems until reaching a base case."
It needs two parts: a base case that stops the recursion, and a recursive case that moves closer to the base.
② Palavras-chave
recursionbase casestackdividefactorial
③ Como Funciona
Each recursive call adds a frame to the call stack. When the base case returns, the stack unwinds, each frame combining its result. Forget the base case and you get a stack overflow.
④ Exemplo de Código
def factorial(n):
    if n <= 1:        # base case
        return 1
    return n * factorial(n - 1)

print(factorial(5))   # 120
function factorial(n) {
  if (n <= 1) return 1;   // base case
  return n * factorial(n - 1);
}
console.log(factorial(5));  // 120
static int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
⑤ Lembre-se
Always write the base case first. Without it, recursion is just an infinite loop with extra steps.

cherry code › N·07
Closure
A function that remembers variables from the scope where it was created.
N·07functionsscope
tags: Estudado Revisar Difícil Favorito
① Definição
"A closure is a function bundled with references to its surrounding state."
When a function is defined inside another function, it can keep using the outer function's variables even after the outer function has returned.
② Palavras-chave
closurescopecapturelexicalenclosing
③ Como Funciona
The inner function "closes over" the outer variables. Each call to the outer function creates a fresh closure with its own copy of the captured state — useful for factories, counters, and private data.
④ Exemplo de Código
def make_counter():
    count = 0
    def inc():
        nonlocal count
        count += 1
        return count
    return inc

c = make_counter()
print(c(), c(), c())   # 1 2 3
function makeCounter() {
  let count = 0;
  return () => ++count;
}
const c = makeCounter();
console.log(c(), c(), c());  // 1 2 3
// Java captures effectively-final vars in lambdas
Supplier<Integer> makeCounter() {
    int[] count = {0};
    return () -> ++count[0];
}
⑤ Lembre-se
A closure is a function plus its backpack — it carries its birthplace's variables wherever it goes.

Default Arguments
Parameter values used when the caller doesn't provide one.
N·09functionsdefaults
tags: Estudado Revisar Difícil Favorito
① Definição
"A default argument is a fallback value assigned to a parameter when no argument is passed."
It makes parameters optional and removes the need for repetitive null checks in callers.
② Palavras-chave
defaultoptionalfallbackoverride
③ Como Funciona
Python and JS let you write def f(x=10) or function f(x = 10). If the caller omits the argument, the default is used. Java has no built-in defaults — you overload methods instead.
④ Exemplo de Código

  return `${greeting}, ${name}!`;
}
greet("Ada");          // Hello, Ada!
greet("Ada", "Hi");    // Hi, Ada!

static String greet(String name) {
    return greet(name, "Hello");
}
static String greet(String name, String g) {
    return g + ", " + name + "!";
}
⑤ Lembre-se
In Python, never use mutable defaults like def f(x=[]). The list is shared across calls and causes infamous bugs.

tags: Estudado Revisar Difícil Favorito

"Variadic parameters let a function accept an arbitrary number of arguments."

② Palavras-chave
*args**kwargsrestvariadicspread

}

    for (int n : nums) sum += n;

The names args and kwargs are convention, not law — what matters are the stars * and **.

cherry code › N·11
Function Scope
Variables declared in a function exist only inside it.
N·11functionsscope
tags: Estudado Revisar Difícil Favorito
① Definição
"Function scope means variables declared inside a function are visible only within that function."
Outside the function, those names don't exist. This keeps internal details from leaking and prevents accidental conflicts.
② Palavras-chave
localglobalscopevisibilitystack frame
③ Como Funciona
When a function is called, a new local namespace is created. When it returns, that namespace is discarded. Variables defined outside the function (global) are visible inside unless shadowed by a local with the same name.
④ Exemplo de Código
def demo():
    secret = 42           # local
    print(secret)

demo()
print(secret)              # NameError
function demo() {
  let secret = 42;       // local
  console.log(secret);
}
demo();
console.log(secret);       // ReferenceError
static void demo() {
    int secret = 42;       // local
    System.out.println(secret);
}
// secret not visible here
⑤ Lembre-se
Locals beat globals. Reaching out to a global variable from inside a function is a sign your design needs work.

    this.right = null;
