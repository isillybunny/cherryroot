Programming
Escrever instruções que um computador consegue executar.
F·01programmingbasics
tags: Estudado Revisar Difícil Favorito
① Definição
"Programar é dizer ao computador o que fazer, passo a passo."
Código é uma sequência precisa de instruções numa linguagem que o computador consegue interpretar. Todo app, site e jogo é, no fundo, uma série de instruções de programação.
② Palavras-chave
codeinstructionslogicexecutionalgorithm
③ Como Funciona
Você escreve o código → o computador lê → executa os passos um a um. Programas resolvem problemas quebrando-os em instruções pequenas e ordenadas.
④ Exemplo de Código
# Your first program
print("Hello, World!")
// Your first program
console.log("Hello, World!");
public class Main {
  public static void main(String[] args) {
    System.out.println("Hello, World!");
  }
}
⑤ Lembre-se
Código é comunicação — entre você e o computador, E entre você e o você do futuro (ou sua equipe).

Code
O texto que escrevemos numa linguagem de programação.
F·02codesource
tags: Estudado Revisar Difícil Favorito
① Definição
"Código é o texto de um programa — escrito numa linguagem específica com regras estritas."
O código-fonte é a versão legível por humanos de um programa. Ferramentas (interpretadores/compiladores) convertem isso em instruções que a máquina realmente roda. Código vive em arquivos com extensões como .py, .js, .java.
② Palavras-chave
source.py.js.javasyntaxlanguage
③ Como Funciona
Você escreve código num editor de texto → salva como um arquivo → roda com a ferramenta certa. Código pode ser uma única linha ou milhões de linhas espalhadas por milhares de arquivos.
④ Exemplo de Código
# File: greet.py
name = "Cherry"
print(f"Hello, {name}!")
// File: greet.js
const name = "Cherry";
console.log(`Hello, ${name}!`);
// File: Greet.java
String name = "Cherry";
System.out.println("Hello, " + name);
⑤ Lembre-se
Código é texto. Só isso. Arquivos de texto que seguem regras estritas.

cherry code › F·03
Compiler vs Interpreter
Duas formas de transformar código em algo que o computador roda.
F·03compilerinterpreter
tags: Estudado Revisar Difícil Favorito
① Definição
"Compiladores traduzem tudo antes de rodar. Interpretadores traduzem linha por linha conforme avançam."
Um compiler converte o programa inteiro em código de máquina de uma vez, depois roda o resultado (rápido, mas demora pra começar). Um interpreter lê e executa linha por linha (mais lento em runtime, mas com feedback instantâneo). Java é compilado. Python e JavaScript são interpretados.
② Palavras-chave
compileinterpretbytecoderuntimeJIT
③ Como Funciona
Compiler: fonte → arquivo executável → roda.
Interpreter: fonte → traduzido E executado linha por linha.
Linguagens modernas misturam os dois (compilação JIT).
④ Exemplo de Código
# Interpreted — run directly
# $ python script.py
print("runs line by line")
// Interpreted by the browser/Node
// No compile step needed
console.log("runs immediately");
// Compiled FIRST: javac Main.java
// THEN run: java Main
System.out.println("compiled then run");
⑤ Lembre-se
Python/JS = digitar e rodar. Java = compilar, depois rodar. No fim, os dois acabam fazendo a mesma coisa.

cherry code › F·04
Bug
Quando o código não faz o que você queria.
F·04bugerror
tags: Estudado Revisar Difícil Favorito
① Definição
"Um bug é qualquer erro no código que causa comportamento errado ou travamentos."
Bugs são inevitáveis — todo programador cria bugs. A habilidade não é evitá-los, mas ACHAR e CORRIGIR rápido. Tipos: erros de sintaxe (nem rodam), erros de runtime (travam no meio da execução), bugs de lógica (rodam, mas com saída errada).
② Palavras-chave
bugerrorcrashlogicedge case
③ Como Funciona
Você escreveu algo do jeito que pretendia — mas o computador lê exatamente o que você digitou. Bug = a distância entre o que você digitou e o que você quis dizer.
④ Exemplo de Código
# BUG: wrong operator
age = 18
if age = 18:  # SyntaxError: should be ==
    print("adult")
// BUG: typo causes ReferenceError
let name = "Cherry";
console.log(nme);  // nme is not defined
// BUG: array out of bounds
int[] nums = {1, 2, 3};
System.out.println(nums[5]);  // crashes!
⑤ Lembre-se
Todo dev sênior ainda cria bugs diariamente. Eles só ficaram mais rápidos em achá-los.

cherry code › F·05
Debug
Achar e corrigir bugs.
F·05debugfix
tags: Estudado Revisar Difícil Favorito
① Definição
"Debugar é o processo de investigar POR QUE o código se comporta mal e corrigir isso."
Passos: 1) Reproduzir o bug de forma consistente. 2) Ler a mensagem de erro (ela costuma dizer exatamente o que está errado). 3) Imprimir valores intermediários pra achar onde a lógica diverge. 4) Corrigir e testar. A maior parte do tempo programando é debugar, não escrever.
② Palavras-chave
debugbreakpointlogtracereproduce
③ Como Funciona
Adicione prints/logs ao redor do código suspeito → rode → veja quais valores realmente passam por ali → encontre a diferença em relação ao que você esperava → corrija.
④ Exemplo de Código
# Add print to debug
result = total / count
print(f"DEBUG: total={total}, count={count}")
# Oh! count is 0 → ZeroDivisionError
// console.log() is your friend
const result = total / count;
console.log("DEBUG:", { total, count });
// → { total: 100, count: 0 }
// Use println for quick debug
System.out.println("DEBUG total: " + total);
System.out.println("DEBUG count: " + count);
⑤ Lembre-se
Leia a mensagem de erro PRIMEIRO. É a frase mais importante na tela.

IDE
Ambiente de Desenvolvimento Integrado — sua oficina de código.
F·06IDEeditor
tags: Estudado Revisar Difícil Favorito
① Definição
"Uma IDE é um software que reúne as ferramentas que você precisa pra programar: editor, debugger, terminal, explorador de arquivos."
IDEs populares: VS Code (a mais popular, grátis, todas as linguagens), PyCharm (Python), IntelliJ (Java), WebStorm (JS). Todas compartilham: destaque de código, autocomplete, verificação de erros, debugger, terminal integrado.
② Palavras-chave
VS CodePyCharmIntelliJeditorautocomplete
③ Como Funciona
A IDE destaca a sintaxe, sugere completações enquanto você digita, marca erros em tempo real e deixa você rodar/debugar sem sair da janela. Opcional, mas transformadora pra produtividade.
④ Exemplo de Código
# In VS Code with Python extension:
# - Auto-complete as you type
# - Inline error: missing :
# - F5 to run, F9 for breakpoint
// VS Code shows:
// - red squiggle on typos

    if (n <= 1) return 1;

Array
Uma sequência ordenada de tamanho fixo, acessada por índice.
S·01datasequence
tags: Estudado Revisar Difícil Favorito
① Definição
"Um array é um bloco contíguo de memória que guarda um número fixo de elementos do mesmo tipo."
Você acessa os elementos em tempo constante com um índice numérico que começa em 0 na maioria das linguagens.
② Palavras-chave
arrayindexlengthfixed sizeO(1) access
③ Como Funciona
Como os elementos ficam guardados lado a lado na memória, o endereço do elemento i é só base + i * size. É por isso que o acesso por índice é tempo constante. Inserir no meio, porém, exige deslocar os elementos.
④ Exemplo de Código
# Python uses 'list' for arrays
arr = [10, 20, 30, 40]
print(arr[0])      # 10
print(len(arr))    # 4
const arr = [10, 20, 30, 40];
console.log(arr[0]);    // 10
console.log(arr.length); // 4
int[] arr = {10, 20, 30, 40};
System.out.println(arr[0]);     // 10
System.out.println(arr.length); // 4
⑤ Lembre-se
Os índices começam em 0 em quase toda linguagem. O último índice é length - 1.

List
Uma coleção ordenada, dinâmica e redimensionável de itens.
S·02datadynamic
tags: Estudado Revisar Difícil Favorito
① Definição
"Uma list é um array dinâmico — uma coleção ordenada que pode crescer ou encolher em runtime."
Lists suportam adicionar, remover e inserir itens em qualquer posição. São a estrutura de dados de uso diário, a faz-tudo.
② Palavras-chave
listappendpopinsertArrayList
③ Como Funciona
Por baixo dos panos, uma list normalmente é sustentada por um array que o runtime redimensiona conforme necessário (dobrando sua capacidade). Anexar no fim é O(1) amortizado; inserir no meio é O(n).
④ Exemplo de Código
items = ["a", "b"]
items.append("c")
items.insert(0, "z")
items.pop()
print(items)   # ['z', 'a', 'b']
const items = ["a", "b"];
items.push("c");
items.unshift("z");
items.pop();
console.log(items);  // ['z', 'a', 'b']
List<String> items = new ArrayList<>();
items.add("a");
items.add("b");
items.add(0, "z");
items.remove(items.size() - 1);
⑤ Lembre-se
Anexe no fim sempre que puder. Inserir no índice 0 de uma list longa é lento — desloca tudo.

Tuple
Um grupo de valores ordenado e imutável.
S·03dataimmutable
tags: Estudado Revisar Difícil Favorito
① Definição
"Uma tuple é uma sequência ordenada de tamanho fixo que não pode ser alterada depois de criada."
Tuples são muito usadas pra agrupar valores relacionados, como coordenadas ou os múltiplos valores de retorno de uma função.
② Palavras-chave
tupleimmutablefixedpairrecord
③ Como Funciona
Python tem um tipo tuple dedicado com a sintaxe (a, b). JavaScript e Java fingem isso com arrays ou pequenos records. Como tuples são imutáveis, elas podem ser usadas como chaves de dictionary no Python.
④ Exemplo de Código
point = (3, 4)
x, y = point        # unpacking
print(x, y)         # 3 4
# point[0] = 9       # TypeError — immutable
// JS uses arrays + Object.freeze for tuples
const point = Object.freeze([3, 4]);
const [x, y] = point;
console.log(x, y);   // 3 4
// Records (Java 14+) act like tuples
record Point(int x, int y) {}
Point p = new Point(3, 4);
System.out.println(p.x() + "," + p.y());
⑤ Lembre-se
Tuples gritam "isso não vai mudar". Elas são pra pacotes fixos de dados, não pra coleções que crescem.

Dictionary / Map
Uma coleção de pares chave → valor com busca rápida.
S·04datamap
tags: Estudado Revisar Difícil Favorito
① Definição
"Um dictionary (ou map) guarda valores indexados por chaves únicas em vez de posições inteiras."
Você busca os dados pelo nome da chave, obtendo acesso de tempo médio O(1) tanto pra leituras quanto pra escritas.
② Palavras-chave
dictmapkeyvaluehash
③ Como Funciona
Internamente, o dictionary computa um hash da chave pra encontrar seu slot num array de apoio. Duas chaves diferentes podem dar hash no mesmo slot (uma colisão), que a implementação resolve de forma transparente.
④ Exemplo de Código
user = {"name": "Ada", "age": 30}
print(user["name"])     # Ada
user["city"] = "Rio"
for k, v in user.items():
    print(k, v)
const user = { name: "Ada", age: 30 };
console.log(user.name);     // Ada
user.city = "Rio";
for (const [k, v] of Object.entries(user)) {
  console.log(k, v);
}
Map<String, Object> user = new HashMap<>();
user.put("name", "Ada");
user.put("age", 30);
System.out.println(user.get("name"));
⑤ Lembre-se
As chaves precisam ser hashable. No Python você pode usar strings, números ou tuples — mas não lists.

Set
Uma coleção não ordenada de itens únicos.
S·05dataunique
tags: Estudado Revisar Difícil Favorito
① Definição
"Um set é uma coleção que guarda cada valor no máximo uma vez e não tem ordem definida."
Sets são perfeitos pra testes de pertencimento e pra remover duplicatas dos dados.
② Palavras-chave
setuniqueunionintersectionmembership
③ Como Funciona
Sets normalmente são implementados com uma hash table, igual aos dictionaries, mas guardando só as chaves. Adicionar uma duplicata não faz nada. Operações de conjunto como união e interseção já vêm prontas.
④ Exemplo de Código
tags = {"a", "b", "a"}
print(tags)              # {'a', 'b'}
tags.add("c")
print("a" in tags)        # True
print(tags | {"d"})      # union
const tags = new Set(["a", "b", "a"]);
console.log(tags.size);       // 2
tags.add("c");
console.log(tags.has("a"));   // true
Set<String> tags = new HashSet<>();
tags.add("a"); tags.add("b"); tags.add("a");
System.out.println(tags.size());    // 2
System.out.println(tags.contains("a"));
⑤ Lembre-se
Pra remover duplicatas de uma list, envolva ela em um set e volte pra list: list(set(items)).

Stack
Coleção LIFO — último a entrar, primeiro a sair.
S·06dataLIFO
tags: Estudado Revisar Difícil Favorito
① Definição
"Uma stack é uma coleção Last-In-First-Out (LIFO): o item adicionado mais recentemente é o primeiro a ser removido."
Pense numa pilha de pratos — você tira de cima. As duas operações principais são push (adicionar) e pop (remover).
② Palavras-chave
pushpoppeekLIFOcall stack
③ Como Funciona
Stacks alimentam as chamadas de função (a call stack), o histórico de desfazer e a análise de expressões. Tanto push quanto pop são O(1). Você pode implementar uma com um array e um pointer pro topo.
④ Exemplo de Código
stack = []
stack.append(1)
stack.append(2)
stack.append(3)
print(stack.pop())   # 3
print(stack.pop())   # 2
const stack = [];
stack.push(1);
stack.push(2);
stack.push(3);
console.log(stack.pop());  // 3
console.log(stack.pop());  // 2
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);
stack.push(2);
stack.push(3);
System.out.println(stack.pop());  // 3
⑤ Lembre-se
Toda chamada de função que você faz usa uma stack — o histórico do seu código é literalmente uma pilha de frames.

Queue
Coleção FIFO — primeiro a entrar, primeiro a sair.
S·07dataFIFO
tags: Estudado Revisar Difícil Favorito
① Definição
"Uma queue é uma coleção First-In-First-Out (FIFO): os itens são removidos na mesma ordem em que foram adicionados."
Como uma fila no caixa — o primeiro a chegar é o primeiro a ser atendido. As operações são enqueue e dequeue.
② Palavras-chave
enqueuedequeueFIFOdequebuffer
③ Como Funciona
Queues alimentam o agendamento de tarefas, barramentos de mensagens e a busca em largura. As implementações usam uma linked list ou um array de duas pontas (deque) pra manter as duas extremidades em O(1).
④ Exemplo de Código
from collections import deque
q = deque()
q.append("a")
q.append("b")
print(q.popleft())   # 'a'
print(q.popleft())   # 'b'
const q = [];
q.push("a");
q.push("b");
console.log(q.shift());  // 'a'
console.log(q.shift());  // 'b'
Queue<String> q = new ArrayDeque<>();
q.offer("a");
q.offer("b");
System.out.println(q.poll());  // 'a'
⑤ Lembre-se
Não use list.pop(0) no Python — é O(n). Use collections.deque pra queues de verdade.

Linked List
Uma corrente de nós onde cada nó aponta pro próximo.
S·08datanodes
tags: Estudado Revisar Difícil Favorito
① Definição
"Uma linked list é uma sequência de nós onde cada nó contém um valor e um pointer pro próximo nó."
Ao contrário de um array, os nós ficam espalhados pela memória. Inserção e remoção em posições conhecidas são O(1), mas a busca por índice é O(n).
② Palavras-chave
nodenextheadtailpointer
③ Como Funciona
Uma linked list simples tem um único pointer next por nó. Uma linked list duplamente ligada também tem um pointer prev. Pra encontrar o elemento i, você precisa caminhar desde a cabeça, um nó de cada vez.
④ Exemplo de Código
class Node:
    def __init__(self, value, nxt=None):
        self.value = value
        self.next = nxt

head = Node(1, Node(2, Node(3)))
print(head.next.value)   # 2
class Node {
  constructor(value, next = null) {
    this.value = value;
    this.next = next;
  }
}
const head = new Node(1, new Node(2, new Node(3)));
class Node {
    int value;
    Node next;
    Node(int v) { value = v; }
}
Node head = new Node(1);
head.next = new Node(2);
⑤ Lembre-se
Linked lists são famosas em entrevistas, raras no código real. A maioria das linguagens prefere arrays dinâmicos pro dia a dia.

Tree
Uma estrutura hierárquica de nós com ligações pai-filho.
S·09datahierarchy
tags: Estudado Revisar Difícil Favorito
① Definição
"Uma tree é uma estrutura de dados hierárquica com um nó raiz e nós filhos ramificando abaixo."
Cada nó tem zero ou mais filhos e exatamente um pai (exceto a raiz, que não tem nenhum).
② Palavras-chave
rootparentchildleafbinary tree
③ Como Funciona
Trees modelam hierarquias — sistemas de arquivos, o DOM, árvores de decisão. Uma binary tree limita cada nó a no máximo dois filhos. As travessias (in-order, pre-order, post-order) visitam cada nó numa sequência específica.
④ Exemplo de Código
class TreeNode:
    def __init__(self, v):
        self.value = v
        self.left = self.right = None

root = TreeNode(1)
root.left = TreeNode(2)
root.right = TreeNode(3)
class TreeNode {
  constructor(v) {
    this.value = v;
    this.left = null;
    this.right = null;
  }
}
const root = new TreeNode(1);
root.left = new TreeNode(2);
class TreeNode {
    int value;
    TreeNode left, right;
    TreeNode(int v) { value = v; }
}
TreeNode root = new TreeNode(1);
⑤ Lembre-se
Pastas, árvores genealógicas, o DOM, JSON — a maioria das hierarquias que você encontra são trees disfarçadas.

Hash Table
O motor por trás da busca rápida por chave-valor.
S·11datahash
tags: Estudado Revisar Difícil Favorito
① Definição
"Uma hash table guarda pares chave-valor num array, usando uma função de hash pra mapear cada chave a um índice."
É a implementação por trás dos dicts do Python, dos objects do JavaScript e dos HashMaps do Java.
② Palavras-chave
hash functionbucketcollisionload factorrehash
③ Como Funciona
A função de hash transforma uma chave num número, depois pega esse número módulo o tamanho do array pra escolher um slot. As colisões são resolvidas com chaining (uma list por slot) ou open addressing (procura no próximo slot).
④ Exemplo de Código
# dict is a hash table
print(hash("hello"))   # big integer
table = {"hello": 1, "world": 2}
print(table["hello"])   # O(1) average
// Object and Map are hash tables
const table = new Map();
table.set("hello", 1);
table.set("world", 2);
console.log(table.get("hello"));
HashMap<String, Integer> table = new HashMap<>();
table.put("hello", 1);
table.put("world", 2);
System.out.println(table.get("hello"));
⑤ Lembre-se
A média O(1) é o sonho. Com um hash ruim, o pior caso vira O(n) — colisões importam.

Big O Notation
Como o tempo ou o espaço de um algoritmo cresce conforme o tamanho da entrada cresce.
S·12datacomplexity
tags: Estudado Revisar Difícil Favorito
① Definição
"Big O descreve como o uso de recursos de um algoritmo escala com o tamanho da entrada."
É a linguagem pra falar sobre eficiência. Classes comuns: O(1), O(log n), O(n), O(n log n), O(n²), O(2ⁿ).
② Palavras-chave
O(1)O(n)O(log n)O(n²)complexity
③ Como Funciona
Big O ignora constantes e termos de ordem menor — ele mostra o formato do crescimento. Um algoritmo O(n²) fica lento de forma quadrática, então dobrar a entrada quadruplica o trabalho. O(log n) mal se abala.
④ Exemplo de Código
# O(1) — constant time
def first(arr): return arr[0]

# O(n) — linear time
def total(arr):
    s = 0
    for x in arr: s += x
    return s

# O(n²) — nested loops
def pairs(arr):
    for a in arr:
        for b in arr:
            print(a, b)
// O(1)
const first = arr => arr[0];

// O(n)
const total = arr => arr.reduce((a, b) => a + b, 0);

// O(n²)
const pairs = arr => {
  for (const a of arr)
    for (const b of arr) console.log(a, b);
};
// O(1)
int first(int[] arr) { return arr[0]; }

// O(n)
int total(int[] arr) {
    int s = 0;
    for (int x : arr) s += x;
    return s;
}
⑤ Lembre-se
O(n²) dentro de um loop com um milhão de itens é um desastre. Pergunte sempre: como isso escala?

B·04styleDRY

① Definição
"DRY — Don't Repeat Yourself — significa que cada pedaço de conhecimento em um sistema deve ter uma única fonte definitiva."

SOLID Principles
Cinco princípios de design OOP pra um código mais fácil de manter.
B·05styledesign
tags: Estudado Revisar Difícil Favorito
① Definição
"SOLID é um acrônimo pra cinco princípios de design OOP que produzem sistemas flexíveis e fáceis de manter."
Single responsibility (responsabilidade única), Open-closed (aberto-fechado), Liskov substitution (substituição de Liskov), Interface segregation (segregação de interface), Dependency inversion (inversão de dependência).
② Palavras-chave
SRPOCPLSPISPDIP
③ Como Funciona
S: uma classe tem um único motivo pra mudar. O: aberta pra extensão, fechada pra modificação. L: subtipos devem poder ser usados no lugar da base. I: interfaces pequenas e focadas em vez de grandes. D: dependa de abstrações, não de implementações concretas.
④ Exemplo de Código
# S — one responsibility per class
class Invoice:
    def total(self): pass

class InvoicePrinter:
    def print(self, inv): pass

# Two reasons to change → two classes
// D — depend on abstractions
class Notifier {
  constructor(sender) { this.sender = sender; }
  send(msg) { this.sender.send(msg); }
}
// sender can be Email, SMS, Slack...
// I — small focused interfaces
interface Readable  { String read(); }
interface Writable  { void  write(String s); }
// Don't force one big interface on everyone
⑤ Lembre-se
SOLID é uma bússola, não uma checklist. Recorra a ele quando um sistema parecer rígido ou frágil.
